User Client Rights List

Admin >> User Setup >> User Client Relation

The User Client Rights List enables you to define a relationship between a user and a client,and assigning rights to the user to switch across clients of a corporation to which the user belongs. This enables the user to initiate transactions for more than one client by switching across to the client to which the user has been assigned rights.

IMP: Ensure that you select the client for which you want to set up the  definition using Switch Client.

Note: The user can  switch across clients only under its parent corporation.

To define a User Client Rights Relationship:

1.      Enter the following filter values,

§        Assign Client Code: Select the client code with which you want to associate the user.

§        User: Select the user code for which you want to set up a user-client relationship.

Note: Only those user codes are displayed in the list for which the “Across Client” check box in the User Entry master is selected.

2.      Click the Filter button to retrieve the definition.

3.      Select the Assign check box of the users from the list below, to associate those users to the client. The status of the User entry changes from Not Assigned to New.

This indicates that a relationship is set between the users and the client and it awaits authorization. Against every such relation are options to enable, disable or discard that user account definition.

Note:  In case you select a new  user, repeat the whole process to  set up the user-client relationship.

To enable the User Client Rights definition, it is mandatory to authorize the requests based on the Maker-Checker concept. The checker can view the new definitions in the User Client Rights Authorization List screen and authorize the same provided the rights to authorize are assigned to that checker.

On successful authorization, the User Client Rights definition can be viewed in the User Client Rights List screen with the status as Authorized.
