Virtual Account Issuance Entry

The Virtual Account Issuance Entry allows you to set up Virtual Accounts.

1.      Specify the following details to add Virtual Accounts:

§        Issuance Number: This is a unique system generated number for each new request and is non-editable.

§        Issuance Date: The date is defaulted to current application date for a new request or date of entry in case of entered record

§        Client Code: This field is auto populated with the clients user’s own code.

§        Description: This is a non mandatory field that allows you to store any description regarding the Virtual Accounts.

§        Client Credit A/c Number: Click the Seek List to select the Client code. The codes available in the Seek List are valid client codes from client master that uniquely identify the client throughout the system.

§         Account Branch: This field is auto populated with the Branch code of the Client account selected and is non-editable.

§        Available length: This field is auto populated and is non-editable.

§        Total Count of Virtual Account Numbers: Type the total number of Virtual Accounts to be issued.

Series

§        Start Number: This field is auto populated based on the Total Count of Virtual Accounts specified. It is editable only in case of full match; else it is based on last issued number incremented by 1.

Note: Start account number is the last number issued for a previous record using the issuance screen. In case there are some intermediate numbers allotted, then it will not allow you to save the record.

All virtual account numbers are unique in GCP CashWeb at any point of time.

§        End No.: This field is auto populated and derived from the Start number + Total Count of New Virtual Accounts.

§        Sample Start Number:This field signifies the sample of the complete (Part A+ Part B+ Part C) virtual account number assigned to client and credit account number combination. This field helps customer users to identify the sample virtual account number before distributing to their payers.

§        Remarks: Type any remarks related to the entry.

2.      Click Save to save the entry. GCP CashWeb displays a message informing you that the entry has been successfully saved.

3.      Click Back to return to the Virtual Account Issuance List screen. The newly added entry is displayed with its status as New and the same must be authorized to become a valid entry.

IMP: These Virtual Account Numbers would be operational only upon authorization and will not be editable thereafter.
