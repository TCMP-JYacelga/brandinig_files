Payment My Product Master Entry

This screen allows you to associate the Bank products offered, to your own My Product and also define attributes and workflow rules like authorization pattern, user access rights, authorization type, transaction type, etc.

1.      To add a new My Product, type the necessary information in the fields provided under the following heads:

§        My Product

Product Details

□          My Product Code: Type a code that will uniquely identify the MyProduct across the system.

□         Description: Type a description for the product

Note: The system does not support the following special characters: “ \ ~ ^ and ‘ in above text fields

□         UDE (Universal Data Entry)Bank Product: Select the corresponding product from the drop-down list. This step maps your product with the product defined at the bank's end.

□         My Product Usage: Select the payment product from the options- Quick Pay or Batch Pay

Configuration Details

□         Authorization Type: This field determines whether the approval for the specified product must be based on Product or Account. The available options are:

My Product: Select this option if you want the approval to be based on Product .

Account: Select this option if you want the approval to be based on a My Product and Account combination.

Maker Checker: Select this option if you want to approve the foreign currency transactions.

Note: Batch split logic will be applicable only for My Product and Account type of authorization. For Maker-Checker type of authorization, Batch split logic will not be applicable.

□         Transaction Type: Select the appropriate transaction type for the product. The available options are:

Simple: This option enables you to authorize the transactions and send them for processing at Batch level. However, the instruments within a Batch have to be of the same product. If you select this option, the client is debited with the total Batch amount and not with the individual instrument amount.

E.g. If there are 2 instruments for 10 INR and 15 INR each, the client will be debited for a consolidated amount of 25 INR.

Flexible: This option enables you to authorize and send each transaction individually at an Instrument level or all transactions together at Batch level. This is the default option. If you select this option, the client is debited with the individual instrument amount.

E.g. If there are 2 instruments for 10 INR and 15 INR each, the client will be debited with 10 and 15 INR separately.

For quick pay transactions, only Flexible transaction type is applicable

□         Authorization Level: Select 'Instrument Level' in case you require authorization for each individual instrument. Select Batch Level for authorization at batch level.

E.g.:

-         Consider there are 2 instruments within a file with amount 1000 & 5000 each. AVM setup is as follows:

0-1000 User 1

1001-10000 User 2

-         If Authorization Level is at 'Instrument Level', then instrument of amount 1000 will go to User 1 for authorization. Instrument of amount 5000 will go to User 2.

-         If Authorization Level is at 'Batch Level', then total batch amount 6000 will go to User 2 for authorization.

 

Note:

-          If Transaction Type is 'Simple' then Authorization Level will always be at ‘Batch Level’.

-         For quick pay transactions, Authorization Level will always be at ‘Instrument Level’.

-         If Authorization Type is 'Account' and Transaction Type is ‘Simple’, then Authorization Level can be set to 'Batch' or 'Batch-Max Instrument Amount'.

-         If Authorization Type is 'Account' and Transaction Type is ‘Flexible’, then Authorization Level can be set to 'Instrument' or 'Batch-Max Instrument Amount

□         Confidential Indicator: Check this box to keep Beneficiary details confidential

□         Ordering Party: Select this check box if you want to allow an ordering party in the transaction initiation for this product.

□         Batch- Max Instrument Amount: This checkbox is enabled only when Authorization Type is set to 'Batch level'. Select this checkbox to consider the max instrument amount within a batch for splitting the batch into AVM/SVM slabs.

E.g.:

-         Consider the Authorization Type is set to 'Batch level'. The file contains 6 instruments with amount 1000, 2000, 3000, 4000, 5000 & 10000. AVM setup is as follows:

0-1000 User 1

1001-10000 User 2

10000- ~ User 3

-         Case 1: Batch-Max Instrument Amount checkbox is unchecked.

In this case, system will total the instrument amount to 25000 and put the batch into last slab i.e. 10001- ~ for authorization.

-         Case 2: Batch-Max Instrument Amount checkbox is checked.

In this case, system will consider the maximum instrument amount within the batch i.e. 10000 and put the batch into slab 1001-10000 for authorization.

Adhoc Details

□         Adhoc Beneficiary: Select the 'Allowed' check box if you want to allow adhoc beneficiaries in the transaction initiation for this MyProduct. If you do not select this option, only registered beneficiaries can be involved in the transactions of this product. By default, this option is enabled

□         Adhoc Bene Bank: Select the 'Allowed' check box if you want the adhoc beneficiary banks to participate in the transactions. If you do not select this option, only regular beneficiary banks can be involved in the transactions of this product

□         Adhoc Ordering Party: Select the 'Allowed'  check box if you want to allow adhoc-ordering parties in the transaction initiation for this product. If you do not select this option, only regular ordering parties can be involved in the transactions of this product

§        Workflow

Manual Entry (Enables clients to bring more flexibility to the product definition by controlling the authorization and send process)

□         Approval Required: Select the check box if you want to manually authorize the manually entered transactions.

□         Auto Approval Limit: By default this field will have value 999999999999.99. This value will be specified in Seller Base currency. The specified limit will be checked against the Instrument Level, Batch Level or Batch - Max Instrument Amount for manually entered transactions. If the instrument or batch amount is less than the Auto Approval Limit and if Approval Required checkbox is unchecked then transaction will be auto authorized.  If Approval Required checkbox is checked then transaction will be authorized as per AVM/SVM setup. If valid AVM/SVM setup is not available then system will raise an exception error.

Note: Ensure that the user sets the Auto Approval Limit as per maximum of any slab in AVM/SVM. In case of mismatch, then AVM/SVM slab will prevail.

□         Verification Required: Select this checkbox for adding an additional stage of verification for this product before AVM/SVM authorization for manually entered transactions. With this functionality, Users for whom 'Verify' rights are provided and Verification Required field is enabled, can authorize/reject the transactions before it flows into AVM/SVM workflow stage.

 Note:

-         Verification is always by ONE user only and not many.

-         Verification will be done at same level as defined for Authorization Level field. In case Authorization Level is set to Batch or Batch-Max Instrument amount, then Verification will be at Batch level else at Instrument level.

-         On Submit system will validate the Verification Required field for the derived product. If the checkbox is selected and users have appropriate rights for verification and product account linkages are done, then system will send the transaction to the Verifier for authorization. At this stage the status of the transaction will be 'For Verification'. Once the verifier rejects the transaction it goes to the maker queue for editing and if he authorizes the transaction then it goes into AVM/SVM workflow for further processing. Post verification, the status of the transaction becomes 'For Auth'.

□         Send Required: Select the check box if you want to route the manually entered transactions through the sending option after authorization.

File Upload

□         Submit Required: Select this check box to allow the transactions to be submitted on Save.

□         Allow Transaction Edit: Select this check box to allow uploaded transactions to be edited.  

□         Approval Required: Select the check box if you want to manually authorize the uploaded transactions.

□         Auto Approval Limit: By default this field will have value 999999999999.99. This value will be specified in Seller Base currency. The specified limit will be checked against the Instrument Level, Batch Level or Batch - Max Instrument Amount for uploaded transactions. If the instrument or batch amount is less than the Auto Approval Limit and is Approval Required checkbox is unchecked then transaction will be auto authorized.  If Approval Required checkbox is checked then transaction will be authorized as per AVM/SVM setup. If valid AVM/SVM setup is not available then system will raise an exception error.

 Note: Ensure that the user sets the Auto Approval Limit as per maximum of any slab in AVM/SVM. In case of mismatch, then AVM/SVM slab will prevail.

□         Verification Required: Select this checkbox for adding an additional stage of verification for this product before AVM/SVM authorization for uploaded transactions. This functionality will be available only for those users for whom 'Verify' rights are provided in Categories & Users masters by the bank. With this functionality, Users for whom 'Verify' rights are provided and Verification Required field is enabled, can authorize/reject the transactions before it flows into AVM/SVM workflow stage.

Note:

-         Verification is always by ONE user only (and not many).

-         Verification will be done at same level as defined for Authorization Level field. In case Authorization Level is set to Batch or Batch-Max Instrument amount, then Verification will be at Batch level else at Instrument level.

-         On close system will validate the Verification Required field for the derived product. If the checkbox is selected and users have appropriate rights for verification and product account linkages are done, then system will send the transaction to the Verifier for authorization. At this stage the status of the transaction will be 'For Verification'. Once the verifier rejects the transaction it goes to the maker queue for editing and if he authorizes the transaction then it goes into AVM/SVM workflow for further processing. Post verification, the status of the transaction becomes 'For Auth'.

□         Split File Batch By AVM/SVM: This checkbox is enabled only when Authorization Level is set to ‘Batch Level’ and Batch-Max Instrument Amount is enabled. Select this checkbox if you want to split the Batch into multiple PIRs based on AVM/SVM slab depending on the Authorization Type. If Authorization Type is set to ‘My Product’ then split the Batch on the basis of batch amount and AVM/SVM slab defined at My Product level. If Authorization Type is set to ‘Account’ then split the Batch on the basis of batch amount and AVM/SVM slab defined at Account level.

Note:

-         The Batch Amount needs to be converted into local currency at Card rates to confirm whether it falls into the said AVM/SVM slab.

-         Batch splitting logic will be applicable only if Approval Required field is unchecked, and Auto Approval Limit (File Upload) field is set to default value i.e. 99999999999.99.

-         Currently, system supports only AVM at account level. While introducing support for SVM at account level, we will allow only users (after validating user & account restriction) as part of SVM expression. Category will not be allowed as part of expression.

-         Effect of changes in AVM/SVM after batch split will not affect the already split batch even if it is in reject repair. Reject repair will be on split batch and file level parameter will apply for split payment request rather than whole file. Hence after repair, no further splitting is to be done.

 E.g.:

-         1: Authorization Type is set to ‘My Product’

Consider the upload file ‘batch_split_tst1.txt’ contains 9 instruments with amount 100, 200, 1000, 1300, 1700, 2000, 2500, 2100 & 3000.

Case 1: Consider MYPROD1 with Authorization Type as ‘MyProduct ‘& Authorization Level as ‘Batch’ & Batch – Max Instrument Amount is enabled.  AVM setup is as follows:

0-1000                 User 1

1001-2000      User2

2001- ~              User3

Case 2: Consider MYPROD1 with Authorization Type as ‘MyProduct ‘& Authorization Level as ‘Batch’ & Batch – Max Instrument Amount is enabled.  SVM setup is as follows:

0-1000                 User 1

0-2000                 User2

0- ~                         User3

In above cases, file will be split into 3 payment requests as follows:

Payment Request1:  Ref No ‘batch_split_tst1.txt’, Total Amount of Payment Request = 1300 with Total No. of Instruments = 3. In both the cases, Payment Request 1 will be available to User 1 for authorization since Batch- Max instrument Amount is 1000.

Payment Request2: Ref No ‘batch_split_tst1.txt’, Total Amount of Payment Request = 5000 with Total No. of Instruments = 3. In both the cases, Payment Request 2 will be available to User 2 for authorization since Batch- Max instrument Amount is 2000.

Payment Request3: Ref No ‘batch_split_tst1.txt’, Total Amount of Payment Request = 7600 with Total No. of Instruments = 3. In both the cases, Payment Request 3 will be available to User 3 for authorization since Batch- Max instrument Amount is 3000.

-         2: Authorization Type is set to ‘Account’

Consider the upload file ‘batch_split_tst1.txt’ contains 9 instruments with amount 800 & 300 belonging to Account 1, instrument 1000 belonging to Account 2, instruments 1300, 1700 & 2000 belonging to Account 2, instrument 2100 belonging to Account 1, instruments 2500 & 3000 belonging to Account 2. Consider MYPROD1 with Authorization Type as ‘Account‘& Authorization Level as ‘Batch’ & Batch – Max Instrument Amount is enabled.  AVM setup is as follows:

For Account 1

0-1000                    User1

1001-2000         User2

2001- ~                  User3

For Account 2

0-1000                   User4

1001-2000         User5

2001- ~                 User6

In above case, file will be split into 5 payment requests as follows:

Payment Request1: Ref No ‘batch_split_tst1.txt’,  Total Amount of Payment Request= 1100 with Total No. of Instruments = 2. In this case, Payment Request 1 will be available to User 1 for authorization since Batch- Max instrument Amount is 800.

Payment Request2: Ref No ‘batch_split_tst1.txt’, Total Amount of Payment Request = 1000 with Total No. of Instruments = 1. In this case, Payment Request 2 will be available to User 4 for authorization since Batch- Max instrument Amount is 1000.

Payment Request3: Ref No ‘batch_split_tst1.txt’, Total Amount of Payment Request = 5000 with Total No. of Instruments = 3. In this case, Payment Request  3 will be available to User 5 for authorization since Batch- Max instrument Amount is 2000.

Payment Request4: Ref No ‘batch_split_tst1.txt’, Total Amount of Payment Request= 2100 with Total No. of Instruments = 1. In this case, Payment Request 4 will be available to User 3 for authorization since Batch- Max instrument Amount is 2100.

Payment Request5: Ref No ‘batch_split_tst1.txt’, Total Amount of Payment Request= 5500 with Total No. of Instruments = 2. In this case, Payment Request  5 will be available to User 6 for authorization since Batch- Max instrument Amount is 3000.

□         Send Required: Select the check box if you want to route the uploaded transactions through the sending option after authorization

Standing Instruction-Execution

□         Submit Required, Approval Required, Auto Approval Limit, Verification Required, Send Required: These fields refer to the execution of Standing Instructions. For details, refer to the above section under File Upload  

Validations

□         User Product: Select this check box to generate a relation between the user and the specified product. If you select this check box, the user can view this product at the time of data entry. By default, this check box is selected  

□         Product Account: Select this check box to generate a relation between the account and the specified product. If you select this check box, this product and account will be available at the time of data entry. By default, this check box is selected.  

□         User Account: Select this check box to generate a relation between the user and the account. If you select this check box, the user can view this account at the time of data entry. By default, this check box is selected.

□         Zero Proofing: Select this check box if you want to specify the total number of instruments and total amount at the PIR header level for that product. If you do not select this check box, the total number of instruments and total amount fields are disabled at PIR header level. The number of instruments that can be specified at PIR would be automatically set based on Transaction captured at Instruction level. By default, this check box is selected 

□         Scrap if not Auth or Send: Maker can specify the number of days after which transactions that are unauthorized or not sent to bank for further processing will be scrapped.  

□         Notify User of Scrap before:  Specify the number of days before which the user will be notified about a definition being scrapped

□         Show Global Attribute for Transaction: Select Yes in case you want to allow multi/cross-seller transactions.
