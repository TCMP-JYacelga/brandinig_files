User Account List

Admin >> User Setup >> User Account

The User Account List stores all the user-account relationships that you set up to ensure application security along with details like client, account type, currency and status of each relation. With this relationship, you can assign specific accounts to a user. While initiating transactions, users can select from only those accounts that are assigned to them.

IMP: Ensure that you select the client for which you want to set up the  definition using Switch Client.

To assign Accounts to a User:

1.      Specify the following filter details (The below filters are not mandatory).

§        User: Select the user to which you want to associate the account.

§        Account: Select the account that you want to associate with the user.

§        Acc. Type: Select the type of the account such as Credit, Savings and Debit.

§        CCY: Select the currency in which the account is held.

2.      Click the Filter button to retrieve the definition.

3.      Select the Assign check box of the accounts from the list below, to associate those accounts to the user. The status for those accounts changes from Not Assigned to New.

4.      This indicates that a relationship is set between the user and the accounts and it awaits authorization. Against every such relation are options to enable, disable or discard that user account definition.

Note:  In case you select a new  user, repeat the whole process to assign accounts to that user.

5.      To enable the User Account definition, it is mandatory to authorize the requests based on the Maker-Checker concept.

6.      The checker can view the new definitions in the User Account Authorization List screen and authorize the same provided the rights to authorize are assigned to that checker.

7.      On successful authorization, the User-Account definition can be viewed in the User Account List screen with the status as Authorized.
