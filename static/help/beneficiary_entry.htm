Beneficiary Entry

1.      In the Beneficiary Entry screen enter the personal, bank, account and contact details of the new Beneficiary under the following heads

§        Personal Details

□         Beneficiary Code:Type a unique code that will be used to identify and reference the beneficiary.

□         Name:Type the name of the beneficiary

□         Mobile Number:Type the beneficiary mobile number.

□         E-Mail ID: Type the beneficiary e-mail address.

□         Resident Status: Select the residential status of the beneficiary. The available options are Resident (default) and Non-Resident.

Beneficiary Bank Details

□         Bank Type: The options are Registered Bank (checked by default), and Adhoc Bank. If you select Registered Bank, the below four fields are seen.

□         Bank Branch Search: Click the Seek icon to select the Bank Branch code. The seek list also specifies the Low/ high value membership clearing code of the listed banks.

Note:

-         In case of registered bank and depending upon the set up done by the bank, bank and branch name in the seek list will show the names in local language also.

-         In case of Adhoc bank, the user needs to input the Bank and Branch name in English.

□         The Bank and the Branch fields get auto populated and uneditable. These fields are displayed in English as well as local language if maintained in local language by the bank.

□         Receiver Correspondent BIC: Type the beneficiary BIC code (SWIFT), in case of cross-border funds transfer.

□         In case you select Adhoc Bank for 'Bank Type', you have to specify the Bank and Branch Code, Address, the BIC Code and the Bank Country,or the country where the Bene bank is situated

Beneficiary Account Details

□        Account Number

□        Account Currency

□        IBAN

□         The IBAN code is necessary for cross-border funds transfer

§        Contact Details

□         Name: This field will be non-editable and will be defaulted with the Name captured in Beneficiary Details Tab.

□        Beneficiary Address

□         Country: Select the beneficiary country from drop-down list showing a list of countries as specified by the bank.

□         State: Select the Beneficiary State from the drop-down list that displays the filtered records of the states within the country selected above.

□         City: Select the Beneficiary City from the seek list that displays the filtered records of the City within the State selected above.

□         Post Code: Type the postal code of the mailing destination for the beneficiary.

Notification Details

□         Type the Telephone number of the beneficiary

□         Document Type: In case of individuals, select the document for personal identification from the drop-down list. Available options are Passport / Driving License / PAN / Tax ID.

□         Document ID: Mandatory if Document Type is selected. This is a multilingual field and supports local language. If document type selected is Tax ID, then you can enter the Tax ID provided by Central Tax Authority.

Default Transaction Setup

□         The Bene Default Product, the Default Payout Location (Demand Draft/Cashier Order) and the Default Pickup Branch are as per the values set in the 'Client Masters'.

□         Default Delivery Mode: You can specify the delivery mode from the drop-down list.

2.      Click Save to save the details of the Beneficiary. On saving the entry you are automatically directed back to the Beneficiaries List screen where you can view the newly added beneficiary with its status as 'New', indicating that it is awaiting authorization.

3.      Click Back to return to the Beneficiary List screen without entering a new record.

4.      Beneficiary entry can be set up auto authorization or maker checker authorization from the bank end.

5.      If the beneficiary master entry is set up for auto authorization then the created beneficiary is immediately displayed in the list as authorized.

6.      If the authorization is set up for maker checker then the record is made available in the Beneficiary Authorization List screen and you can then Authorize the same.
