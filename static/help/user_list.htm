User List

Admin >> User Setup >> User

A user of Global CASHplus-ANZ is any individual who interacts with the application to perform various tasks.  You can modify user information by altering the rights that have been assigned to that User. To modify a user, either the existing rights can be revoked or new ones can be granted. The renewed rights will have to be applied to each user within that category to maintain consistency, if required. As a user of the application, you will not be able to edit information of the category to which you belong.

The User List screen displays a summary of the existing users defined in Global CASHplus GCP CashWeb., their description and their status.

□         Filter Criteria: Filter by User Code to retrieve a particular User

□         Code: The User Code is a unique value used to reference that user throughout the system.

□         Description: View the user defined description.

□         Status: View the status of the user.

□         History/ View/ Edit/ Enable/ Disable/ Discard- refer to Other Possible Tasks

IMP: Ensure that you select the client for which you want to set up the  definition using Switch Client.  

For every User that will be logging on to GCP CashWeb, you have to define a User ID.

  • Once you create a User ID you assign the user to a category. The user inherits all the access rights of this category.

  • You also have an option to increase or reduce the access rights of the user, regardless of the access rights of the category to which the user belongs.

  • When the user logs on to Global CASHplus GCP CashWeb Client Console, the login information is internally validated against the information for this User ID that you create here. On a successful logon, the user can perform the assigned tasks.

Defining a User ID:

1.      Click Add User to go to the User Master Entry screen where you can set up a new User ID.

2.      The newly defined User is added to the User List, with its state specified as New and the same can be edited or discarded

Note: If you have created users that you do not want to enable for further use in the system and if those users are not yet authorized, you can Discard them.

3.      To enable the User, it is mandatory to authorize it based on the Maker-Checker concept.

4.      The checker can view the new definition in the User Authorization List screen and authorize the same provided the rights to authorize are assigned to that checker.

5.      A successfully authorized User definition request can be viewed in the User List screen with its status as Authorized.

Editing a User:

1.      Click Edit to go to the User Master Entry screen where you can edit a previously defined master.

2.      Click Update to save the changes.

Note:

 You can modify the definitions if they are in any of the following states:

-         New : Any new definition is not yet authorized and is in the new state.

-         Modified : Any definition that you have modified and not yet authorized and is in the Modified state.

-         Authorized: The definition that is authorized and is in the Enabled state.

The edit option is not available for the definitions that have the state as Disabled, Disable Request, Enable Request, Disable Request Rejected, and Enable Request Rejected, Submitted, Authorized.

On saving the modifications, the definition state changes to Modified. To enable the definition with the Modified state, the definition has to be authorized.

Viewing a User:

1.      Click View to go to the User Master Entry screen where you can view a previously defined master. You cannot modify the definition in this screen.

2.      Click Back to return to the User List screen.
