Creating a Transaction Template

Payments >> Payments >> Template >> Template Summary

The Transaction Template benefits the institutional customer of Bank with recurring payment transactions to specified beneficiaries. Customer can store the beneficiary details in a predefined form and save it as a Transaction Template that can be availed of  to initiate a payment transaction. Templates can be created by two methods.

•       From the existing transaction in payment center

•       From the transaction template menu.

From the existing transaction

1.      In the payment summary screen, click  against the transaction that you want to store as a template transaction.

2.      The screen navigates to the Batch Header or Quick Pay Edit screen depending upon the selection

3.      Enter the name of the Template Name to identify the template.

4.      Refer step 4 to 8 of template creation from transaction template menu for further details.

From the Transaction template menu

1.      In the Template Summary Screen, click to open the New Payment popup.

2.      Select the required My Product to create a transaction template. In case you select a ‘My Product’ linked to Batch payment, you are guided to the  ‘Batch Header’ screen, and the transaction templates can be created with multiple instruments (i.e. Batch Template) whereas if the ‘My Product’ is linked to Quick pay, you are guided to the existing ‘Quick Pay Edit’ screen, and the transaction template will be created for a single instrument (Instrument Template).

3.      Define the template with a Template Name (other details are as per the regular Batch Header or Quick Pay Edit screen for payment transactions).

4.      Click Lock Instructions to lock the fields and make them uneditable to the user creating the transaction using this template. Below mentioned fields can be locked while instrument entry.

•    Product

•    Debit Account

•    Payment Amount

•    Beneficiary Code

•    Debit Reference

•    Credit Reference

•    Debit Details

•    Credit Details

•    Payment location

Note: System will apply unique check on the ‘Template Name’ field for both Batch and Quick Pay transaction templates as per following logic.

-         Transactions with status Pending Repair and Deleted status cannot be saved as template.

-         Batch Template: Batch Template Name must be unique across within a client. Further Instrument details within the Template will be validated for uniqueness within a Batch. For a Batch Template, all the details on Batch header must be as per ‘My Product’ Configuration. It is mandatory to enter details of at least one instrument to make it eligible for transaction cloning.

-         Quick Pay Template:  Quick Pay Template Name must be unique within a client (across Batch and Quickpay template names). At Instrument Level (Quick Pay), Beneficiary Details (Registered / Adhoc) are mandatory, while all other details are optional. User has the option to edit an existing template in case of any changes in the beneficiary profile / details etc.

5.      Click to save the template. On successful save, system will display an appropriate message indicating that the template has been saved. In case of validation failure, system will display a warning message indicating the error. However you can save the template with partial information.

6.      You need to authorize the template as per the maker checker concept. An authorized template is automatically associated by default to the users for the given client in the system. The availability of the template will depend upon the user product linkage. If the product set up does not call for user linkage then the template will be available to all the users of the client.

7.      The template once created can be edited and reused after authorization.

8.      The successfully saved template is made available on selecting New From Template option to generate transaction in Payment center. Templates with status as ‘Template Ready’ will be available for cloning to initiate a transaction.

Note: To keep data consistency between the records created manually or Via Copy to Template feature,  system validates as below:

-         User access right for ‘Template Centre’,

-         My Product linked to Batch transaction is in Enabled State and is linked to User copying to Template

-         Debit Account and MyProduct Linkage is in place based upon MyProduct Config.

-         User and MyProduct Linkage is in place based upon MyProduct Config.

-         User and Debit Account Linkage is in place based upon MyProduct Config.
