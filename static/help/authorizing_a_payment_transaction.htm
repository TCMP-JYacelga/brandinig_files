Authorizing a Payment Transaction

About Authorization of Payment Transactions

IMP:

-         All transactions initiated in GCP CashWeb have to be authorized and sent to bank for further processing within a time frame set in the My Product master.  At both these stages, the Client Checker  (when the status of the transaction is Pending My Auth) and the Client, (when the status of the transaction is Pending Release) will receive a notification of the pending action.

-         In case a transaction remains unauthorized, system will notify the client before scrapping the transaction based on the ‘number’ of days entered in the ‘Notify User of Scrap before’ field in the MyProduct master.

-         Based on the ‘Scrap if not Auth or Send’ limit set in the MyProduct master by the Maker, all transactions with status- ‘Pending Auth’ and ‘Pending My Auth’, exceeding the limit will be scrapped bt the system and notification of the same will be sent to the Client and the Client Checker (authorizer). This is applicable for all the GCP CashWeb Client initiated transactions.

Authorizing Batch/ Instrument by Client

Authorizing a Batch/ Instrument by Token Device:

1.      In the Payment Center, select the transactions with status showing Pending My Auth and click to open Select Signature Type popup.

2.      Select Token option and click Continue to go to Batch / Instrument Authorization Summary screen.

3.      The Token Entry Field displays a unique system generated number.

4.      Enter the number generated in the ‘Token Entry Field’ into the Token Device and press the Generate to generate a unique numeric code. This is the secure Digital Signature.

5.      Enter the unique numeric code in the Digital Signature field for the selected transactions.

6.      Click Sign & Submit to complete the authorization process.

 

Authorizing a Batch/ Instrument by Smart Card:

1.      In the Payment Center, select the transactions with status showing Pending My Auth and click to open Select Signature Type popup.

2.      Select Smart Card option and click Continue to go to Batch / Instrument Authorization Summary screen.

3.      In the Batch/ Instrument Authorization screen, the Sign button is enabled when you select the Smart Card option.

4.      Click Sign. The e-Signer box appears.

5.      Enter the appropriate password to authorize the batch/transaction.

Note: The Smart Card instrument  has to be correctly plugged into the User’s computer terminal.

On successful authorization the screen will display a message confirming the same. The status of the authorized instrument changes to - Pending Release which indicates that it is ready to be sent to the bank for further processing. Depending on the parameters set up in the My Products master, the transactions are sent manually or automatically.

To reject or delete the transaction, select the transaction, click More and select Reject or Discard. To reject a transaction, it is mandatory to specify a reject reason.

Note:

-         All the discarded transactions can be viewed in the summary screen with the status as Deleted and are available for replicating. Once replicated, they are available  for initiating the transaction.

-           When the status of the transaction changes to 'Pending Release', client will receive a notification of the same. This event will be fired on authorizing the transaction.  In case a transaction with status ‘Pending Release’ does not appear in the summary screen, the same may have been scrapped based on the ‘Scrap if not Auth or Send’ limit set in the MyProduct master. Notification of the same will be sent to the Client and the Client Checker (authorizer) of the same. This is applicable for all the GCP CashWeb Client initiated transactions.

Send Transactions to the Bank

Send’ signifies an action where transactions are sent to the the back office application for further processing, after client has initiated and authorized the transactions in GCP CashWeb. This step signifies submission of payment transaction to bank for further processing.

Depending on the parameters set up in the My Products master, the transactions can be sent manually or automatically.

You cannot send the transactions that are on ‘Hold’. You need to first ‘Release’ the transactions from hold and then ‘Send’ to the bank for further processing. In case you are assigned Across Entity rights for a corporation, you can perform this function for clients spread across different countries.

Note: In case Auto Send is selected at MyProduct level for products which do not require an explicit “Send”, the authorized transactions are automatically transmitted to the back office application for further processing. You can use this option to send transactions which are back / current / future dated. Back / current dated transactions are automatically rolled over to the subsequent working day if they cross the cutoff time. Future dated transaction are subsequently warehoused at GCP CashWeb backend till their due date.

To ‘Send’ Instructions to Bank for Further Processing of Transaction:

1    In the Payment Summary screen, select the record with status ‘Pending Release’. You can only send those transactions for which you are assigned the rights.

2    Click to release the record to the bank for further processing. The status of this transaction now changes to Sent to Bank.

3    In case you wish to discard a transaction at this point, click the . A popup appears where you have to specify the reason for holding or discarding the transaction.

4    Click OK. A message confirming the action appears in the header area of the screen.

Note: All the discarded transactions can be viewed in the Payment Summary screen with the status as Deleted and are available for cloning. Once replicated, they are available for initiating the transaction

5    You also have the option to Hold the transaction.

Note: In case a future-dated transaction on authorization is sent to bank from Client-end, the same will show the status as ‘Warehoused’ instead of Sent to Bank. This ensures the Client is also aware of the pending status of the transaction.

Hold/ Release a Transaction

GCP CashWeb provides a feature for holding a transaction due to reasons such as insufficient funds for making the collections etc. Only those transactions with status ‘Pending Release’ can be put on hold. This prevents the transaction from being released to Bank. It is mandatory to release and authorize such transactions kept on hold for further processing. This feature is available only for authorizers with privileges to this screen.

1    In case you wish to hold a transaction at this point, in the Payment center, select the transaction and click . A popup appears where you have to specify the reason for holding the transaction.

2    Click OK. A message confirming the action appears in the header area of the screen.

3    To release a document kept on hold, select the transaction and click icon

Note: Every release request has to be authorized. As the authorizer, you need to validate the correctness of the information provided for the hold request and either accept the record or reject it.

Stop/ Cancel Payment Request for Instruments initiated in GCP CashWeb

Marking a Cancel / Stop Pay Request for Instruments initiated in GCP CashWeb

Stop payment is an order issued to a bank by clients for not making a payment against an instruction initiated on their account. Stop Payment is done when the instrument is handed over to beneficiary and the Bank / Client is not in possession of the instrument. When the physical instrument is still in possession of the client, the stop pay request is internally referred to as ‘Cancellation’.

After issuance of payment initiation requests, client can subsequently inform the system to stop the payment being made to the beneficiary through the stop payment request. By raising a stop payment request, client can stop the payment instructions for physical as well as electronic instruments.

At any point of time, you, as a maker can view the beneficiary and advice Info details that you have entered for a transaction marked as ‘Confidential’. However, while marking stop payment requests for the ‘Confidential’ instructions; all other users can view these details only if the View Confidential flag has been selected for their user IDs in Application Users master

GCP CashWeb allows you to stop or cancel physical as well as electronic payments which are already initiated in the system using the Stop /Cancel Payments feature, at Instrument level.  A stop request can be marked only after the instruction has been authorized by the bank. In case you are assigned Across Entity rights for a corporation, you can stop transactions from processing further for clients spread across different countries.

Note:

-         Only those payment transactions with status as ‘Sent to Bank’ for electronic instruments and 'Received by Bank' in case of physical instruments, are available for Stop payment in GCP CashWeb.

-         Warehoused transactions can be stopped only once they are active in the system.

To mark a stop/cancel payment:

1    In the Payment center, click the Instrument tab to go to the Instrument summary, select the check box of the transaction for which you want to issue the stop payment instruction. These transactions should necessarily show status as Debited.

2    Click More to open the additional actions that can be applied to a transaction and click Stop.

3    Click to process the stop request. The transaction will now be available for authorization.
