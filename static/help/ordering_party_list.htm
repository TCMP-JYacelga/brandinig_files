Ordering Party List

Payments >> Masters >> Ordering Party  

The Ordering Party List stores the details of all the regular ordering parties of your corporate clients that have been defined in the system, along with their relevant details like name, code, status, etc. Against every account are options to discard, enable or, disable that account definition. The Ordering Party List screen also allows you to set up a new ordering party for which you need to provide the name, address, and contact details.

View the details of the grid under the following heads

□         No.:This is the Serial Number of the ordering party.

□         Enable: If you want to use the disabled definition again in the system, you need to enable it. If accepted on authorization, the definition will be enabled. If rejected on authorization, the rejected request will have to be discarded so as to revert the definition to its Disabled state.

□         Disable: You can disable a definition, if it is not to be used in the system any more

□         Discard: If you have created definitions that you do not want to enable for further use in the system and if those definitions are not yet authorized, you can delete them. Similarly, you can delete the new definitions that are rejected in authorization.

□         Ordering Party Code: This unique code is used to identify and reference the ordering party.

□         Name: Type the name of the ordering party

□         Status: Every ordering party definition is associated with a state that indicates the current action that has been taken on it.

The following are various states that can be associated to a definition depending upon the action performed.  

New: Indicates that the definition has been newly added and has not yet been authorized.

Enabled: Indicates that the definition has been authorized and saved.

Modified: Indicates that an enabled definition that is now being changed, and which has yet to be authorized.

Disabled: Indicates a definition that has been disabled and authorized.

Disable Request: Indicates that an enabled definition has been disabled, but which has not yet been authorized.

Enable Request: Indicates that a disabled definition has been enabled, but for which authorization is pending.

New Reject: Indicates that a new definition has been rejected on authorization. It can either be modified or deleted.

Modify Reject: Indicates that a modified definition has been rejected on authorization. It can either be discarded or edited. If discarded, its status will change to that prior to modification.

Disable Request Rejected: Indicates that the disable request for an enabled definition has been rejected during authorization.

Enable Request Rejected: Indicates that the enable request for a disabled definition has been rejected during authorization.

Note:

-         Click History to view  the Maker-Checker history of the last change done on the Ordering Party definition.

-         Viewthe  details  of the corresponding  definition in the Ordering Party Master View screen, provided you have access rights to view the particular definition.

-         Click Edit to edit  the  Ordering Party  details in the Ordering Party Entry screen and update the entry, if you have the editing rights. You can modify a definition provided you have access rights to the definition entry screen. You can modify the definitions if they are in either of the following states:

-         New definition: Any new definition is not yet authorized and is in the new state.

-         Modified definition: Any definition that you have modified and not yet authorized and is in the Modified state.

-         Enabled definition: The definition that is authorized and is in the Enabled state.

Defining an Ordering Party:

Click Add New Ordering Party to go to the Ordering Party Entry screen to set up the new Ordering Party.

§        A successfully defined ordering party appears in this screen with status as 'New' and if needed you can Edit or Discard the beneficiary.

§        Discarding an unauthorized Ordering Party automatically deletes all the account definitions set up for the same.

§        If you are eligible for authorization, the record will be available in the Ordering Party Authorization List screen and you can then Authorize the same.

§        A successfully authorized record will also appear in this screen with the status - 'Authorized' and you can Enable or Disable it provided you have the rights to do so.

§        In case you do not want to use it in future you can Disable it provided you have the rights to do so.

Note: To add more Ordering Parties, click Add New Ordering Party and then repeat the same process

Editing an Ordering Party Definition:

1.      Click Edit to go to the Ordering Party Entry screen where you can edit a previously defined master.

2.      Click Update to save the changes.

Note:

 You can modify the definitions if they are in any of the following states:

-         New definition: Any new definition is not yet authorized and is in the new state.

-         Modified definition: Any definition that you have modified and not yet authorized and is in the Modified state.

-         Enabled definition: The definition that is authorized and is in the Enabled state.

The edit option is not available for the definitions that have the state as Disabled, Disable Req, Enable Req, Dis. Req. Reject, and Ena. Req. Reject, Submitted, Authorized.

On saving the modifications, the definition state changes to Modified. To enable the definition with the Modified state, the definition has to be authorized.

Viewing an Ordering Party Definition:

1.      Click View to go to the Ordering Party Master View screen where you can view a previously defined master. You cannot modify the definition in this screen.

2.      Click Back to return to the Ordering Party List screen.
