User Reports Rights List

Admin >> User Setup >> User Report

The User Reports Rights List screen enables you to define the rights to a Client User within a corporation to view the reports generated in the system. It also displays the existing Users, Clients, their Reports and their present status.

You can filter by User to retrieve a particular User Report.

IMP: Ensure that you select the client for which you want to set up the  definition using Switch Client.  

To define User-Reports Rights:

1.      Select the User to whom you want to assign the rights to view reports.

2.      Click the Filter button to retrieve the definition.

3.      Select the Assign check box of the reports from the list below, to associate those report to the user.

The status for those accounts changes from Not Assigned to New. This indicates that a relationship is set between the user and the reports and it awaits authorization. Against every such relation are options to enable, disable or discard that user report definition.

Note:  In case you select a new  user, repeat the whole process to assign reports to that user.

4.      To enable the User Reports Right, it is mandatory to authorize the requests based on the Maker-Checker concept.

5.      The checker can view the new definitions in the User Reports Rights Authorization List screen and authorize the same provided the rights to authorize are assigned to that checker.

6.      On successful authorization, the reports can be viewed in the User Reports Rights List screen with the status as Authorized.
