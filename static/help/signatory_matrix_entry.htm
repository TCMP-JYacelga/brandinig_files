Signatory Matrix Entry

Topics Covered

•    Defining an Signatory Matrix

•    Editing an Signatory Matrix Definition

•    Viewing an Signatory Matrix Definition

The Signatory Matrix Entry screen allows you to set limits for each product or account. For each limit, you need to the assign the users who are required to approve the transaction.

To set up a Signatory Matrix definition:

1.      Enter the following details

§        Authorization Parameter: This field is non editable. This field is automatically populated with the value specified in the “authorization Matrix List” screen

§        Product: Select the appropriate product from the drop down list

§        Authorization Type: Displays the category of authorization such as Product or Account as defined for the authorization matrix. In case of Account, you need to select the account number.

§        Currency:  Click and select the currency. Authorization matrix can be based on the currency factor also. Click here to learn more about currency based authorization matrix. This field is defaulted to seller base currency.

§        Limit From and Limit To: Type the lower and the upper transaction limit in the respective fields. This limit is a range of transaction amount that decides the application of the authorization rule. Any transaction initiated will be saved only if it satisfies the specified limit.

§        Start Group and End Group: Click Start Group, if you want to use the logical operator ‘AND’. Click End Group, if you want to complete the usage of the ‘AND’ operator. Any category or user you select hereafter will be followed by an ‘OR’ operator

§        Category: Select the category, if you want any user belonging to this category to authorize the transaction and click Add. The selected category is displayed in the Signatory Matrix box.

§        User: Select the user, if you want any particular user to authorize the transaction and click Add. The selected user is displayed in the Signatory Matrix box.

§        Signatory matrix: The selected Category and User are added to this box.

2.      Click Save to save the definition. The authorization matrix definition is added to the list in the Auth Matrix List screen, where it awaits authorization.

3.      Click Refresh to clear the selected category and user and add new ones.

4.      Click Back to go to the previous page without saving the definition

Editing an Signatory  Matrix Definition:

1.      Click Edit to go to the Signatory  Matrix Entry screen where you can edit a previously defined master.

2.      Click Update to save the changes.

Note:

  You can modify the definitions if they are in any of the following states:

-         New : Any new definition is not yet authorized and is in the new state.

-         Modified : Any definition that you have modified and not yet authorized and is in the Modified state.

-         Authorized: The definition that is authorized and is in the Enabled state.

The edit option is not available for the definitions that have the state as Disabled, Disable Request, Enable Request, Disable Request Rejected, and Enable Request Rejected, Submitted, Authorized.

On saving the modifications, the definition state changes to Modified. To enable the definition with the Modified state, the definition has to be authorized.

Viewing an Signatory  Matrix Definition:

1.      Click View to go to the Signatory  Matrix Master View screen where you can view a previously defined master. You cannot modify the definition in this screen.

2.      Click Back to return to the Authorization Matrix List screen.
