About Collection Transactions

A typical Collection transaction involves a request initiated by the client through data entry to the System Bank branch to credit the drawer a particular sum of money. This request provides the System Bank with the necessary details such as the client’s account, the details of the beneficiary bank and amount to be paid. On the basis of these details, the bank can then process the payment transaction.

To facilitate efficient transaction tracking, GCP CashWeb offers the capability to originate transactions using the Deposit – Instrument concept. A Deposit can contain one or more instruments. Moreover, GCP CashWeb enables you to initiate transactions across products within a Deposit.

IMP.: Before you begin with initiating a collection transaction, ensure that all the required masters have been set up.

The Collection Center displays a list of all the existing deposits along with their relevant details and their status under their respective heads- Deposit and Instrument.

The menu bar shows the filters  that can be applied to retrieve a transaction satisfying the selected criteria, along with the Actions that can be taken on a transaction.

The below listed actions can be performed on transactions:

□         New: Click New to initiate a new transaction or template.

□         Submit: This action is available after a transaction is successfully saved. Click Submit to submit the transaction for verification or authorization.

□         Verify: This action is available after a transaction is submitted; It is based on the Payment MyProduct setup and the rights available to the user. Click Verify to verify the transaction details.

□         Authorize: This action is available on Submit/Verify and the authorization level set in the Payment MyProduct setup. Click Authorize to authorize a transaction.

□         Send: This action is available after a transaction is successfully authorized. Click Send to send the transaction to the bank for further processing.

The below options are listed under More and are related to the above actions:

□         Reject: This option is available to the authorizer to Reject a transaction.

□         Hold: This option is available for a transaction that is authorized but not sent to the bank. Click Hold to hold back the transaction for any particular reason or pending action

□         Release: This option is available only for transactions put on hold. Click Release to release the transaction

□         Stop: This option is available for transactions that are authorized and the status is Pending Release. Click Stop to stop the payment transaction.

□         Discard: This option is available for all transactions with status as Draft and Pending Submit. Click Discard to delete the transaction.

Note: All the discarded transactions can be viewed in the summary screen with the status as Deleted and are available for cloning. Once replicated, they are available in the for initiating the transaction.

□         Download: This option is available for downloading transaction details

□         Print: This option is available for Printing transaction details.

Initiating a Collection Transaction

1.      Click the icon in the Deposit summary screen to open the New Deposit pop-up showing a list of direct debit products.

2.      Select a product, the options are 'Physical' and 'Electronic' (e.g. click the Electronic icon for online transaction) and you are taken to the Deposit screen where you can enter Deposit level details.

3.      Entering Deposit Details

4.      Adding Instruments to the Deposit

5.      Submitting a Transaction:

§        On successful initiation, the transaction will appear in this screen with the status showing - For Submit. Select the record and click the  icon to make it available for authorization. If successfully submitted, system displays a message confirming it and notifies the Client Checker (authorizer) that the transaction has been entered and closed and is awaiting authorization.

Note: In case My Product flag is set to Deposit in the Product master, you can Submit, Authorize and Send a Deposit with multiple instruments in the Collections summary screen

IMP: You can delete an instruction entry within a Deposit only before you Submit the Deposit. Once you Submit a Deposit, you cannot make any changes to it, both at the deposit level as well as instruction level .

§        Based on the Maker-Checker concept, if you are eligible for authorization, the status of the record will show - For My Auth, else, For Auth.

Click Authorizing a Collection Transaction and Releasing it to the Bank to learn more.

Editing a Collection Transaction

1.      Click Edit to go to the Deposit Header screen in case you are at Deposit level, or the Instrument Edit screen if you are at Instrument level, where you can edit a previously initiated transaction.

2.      Click Update to save the changes.

Note:  You can modify a transaction only till such time that it is not Submitted.

Viewing a Collection Transaction:

1.      Click View to go to the Deposit screen in case you are at Deposit level, or the Instrument View screen if you are at Instrument level,where you can view a previously initiated transaction. You cannot modify the transaction in this screen.

2.      Click Back to return to the Collection center.
