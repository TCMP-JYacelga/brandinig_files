Payment My Product Master View

This screen allows you to view the Bank products associated to your own My Product along with the attributes and workflow rules like authorization pattern, user access rights, authorization type, transaction type, etc.

View the details under the following heads:

□         My Product

Product Details

□          My Product Code: View the code that will uniquely identify the MyProduct across the system.

□         Description: View thea description for the product

□         UDE Bank Product: View the corresponding product selected that maps your product with the product defined at bank's end.

□         My Product Usage: View the payment product selected from the options- Quick Pay or Batch Pay

Configuration Details

□         Authorization Type: This field determines whether the approval for the specified product must be based on Product or Account. The available options are:

My Product: This indicated whether you want the approval to be based on Product .

Account: This indicates whether you want the approval to be based on a My Product and Account combination.

Maker Checker: This indicated whether you want to approve the foreign currency transactions.

□         Transaction Type: View the transaction type selected for the product from the below options:

Simple: This option enables you to authorize the transactions and send them for processing at Batch level. However, the instruments within a Batch have to be of the same product. If you have selected this option, the client is debited with the total Batch amount and not with the individual instrument amount.

E.g. If there are 2 instruments for 10 INR and 15 INR each, the client will be debited for a consolidated amount of 25 INR.

Flexible: This option enables you to authorize and send each transaction individually at an Instrument level or all transactions together at Batch level. This is the default option. If you have selected this option, the client is debited with the individual instrument amount.

E.g. If there are 2 instruments for 10 INR and 15 INR each, the client will be debited with 10 and 15 INR separately.

For quick pay transactions, only Flexible transaction type is applicable

□         Authorization Level: 'Instrument Level' indicated whether you require authorization for each individual instrument. 'Batch Leve'l indicates authorization at batch level.

Note:

-          If Transaction Type is 'Simple' then Authorization Level will always be at ‘Batch Level’.

-         For quick pay transactions, Authorization Level will always be at ‘Instrument Level’.

-         If Authorization Type is 'Account' and Transaction Type is ‘Simple’, then Authorization Level can be set to 'Batch' or 'Batch-Max Instrument Amount'.

-         If Authorization Type is 'Account' and Transaction Type is ‘Flexible’, then Authorization Level can be set to 'Instrument' or 'Batch-Max Instrument Amount

□         Confidential Indicator: Selection of this check box indicated whether you what to keep Beneficiary details confidential

□         Ordering Party: This selection indicated whether you want to allow an ordering party in the transaction initiation for this product.

□         Batch- Max Instrument Amount: This checkbox is enabled only when Authorization Type is set to 'Batch level'. Selecting this checkbox allows you to consider the max instrument amount within a batch for splitting the batch into AVM/SVM slabs.

E.g.:

-         Consider the Authorization Type is set to 'Batch level'. The file contains 6 instruments with amount 1000, 2000, 3000, 4000, 5000 & 10000. AVM setup is as follows:

0-1000 User 1

1001-10000 User 2

10000- ~ User 3

-         Case 1: Batch-Max Instrument Amount checkbox is unchecked.

In this case, system will total the instrument amount to 25000 and put the batch into last slab i.e. 10001- ~ for authorization.

-         Case 2: Batch-Max Instrument Amount checkbox is checked.

In this case, system will consider the maximum instrument amount within the batch i.e. 10000 and put the batch into slab 1001-10000 for authorization.

Adhoc Details

□         Adhoc Beneficiary: The 'Allowed' check box indicated whether you want to allow adhoc beneficiaries in the transaction initiation for this MyProduct. If you have not selected this option, only registered beneficiaries will be involved in the transactions of this product. By default, this option is enabled

□         Adhoc Bene Bank: The 'Allowed' check box indicates whether you want the adhoc beneficiary banks to participate in the transactions. If you have not selected this option, only regular beneficiary banks will be involved in the transactions of this product

□         Adhoc Ordering Party: The 'Allowed'  check box indicates whether you want to allow adhoc-ordering parties in the transaction initiation for this product. If you have not selected this option, only regular ordering parties will be involved in the transactions of this product

□         Workflow

Manual Entry (Enables clients to bring more flexibility to the product definition by controlling the authorization and send process)

□         Approval Required: This check box indicates whether you want to manually authorize the manually entered transactions.

□         Auto Approval Limit: View the maximum amount that allows auto approval.

□         Verification Required: This check box allows for adding an additional stage of verification for this product before AVM/SVM authorization for manually entered transactions.

 Note:

-         Verification is always by ONE user only and not many.

-         Verification will be done at same level as defined for Authorization Level field. In case Authorization Level is set to Batch or Batch-Max Instrument amount, then Verification will be at Batch level else at Instrument level.

-         On Submit system will validate the Verification Required field for the derived product. If the checkbox is selected and users have appropriate rights for verification and product account linkages are done, then system will send the transaction to the Verifier for authorization. At this stage the status of the transaction will be 'For Verification'. Once the verifier rejects the transaction it goes to the maker queue for editing and if he authorizes the transaction then it goes into AVM/SVM workflow for further processing. Post verification, the status of the transaction becomes 'For Auth'.

□         Send Required: This check box indicates whether you want to route the manually entered transactions through the sending option after authorization.

File Upload

□         Submit Required: This check box allows the transactions to be submitted on Save.

□         Allow Transaction Edit: This check box allows uploaded transactions to be edited.  

□         Approval Required: The check box indicates whether you want to manually authorize the uploaded transactions.

□         Auto Approval Limit: View the auto approval limit set for this product

□         Verification Required: This checkbox indicated whether an additional stage of verification is required for this product before AVM/SVM authorization for uploaded transactions. With this functionality, Users for whom 'Verify' rights are provided and Verification Required field is enabled, can authorize/reject the transactions before it flows into AVM/SVM workflow stage.

Note:

-         Verification is always by ONE user only (and not many).

-         Verification will be done at same level as defined for Authorization Level field. In case Authorization Level is set to Batch or Batch-Max Instrument amount, then Verification will be at Batch level else at Instrument level.

-         On close system will validate the Verification Required field for the derived product. If the checkbox is selected and users have appropriate rights for verification and product account linkages are done, then system will send the transaction to the Verifier for authorization. At this stage the status of the transaction will be 'For Verification'. Once the verifier rejects the transaction it goes to the maker queue for editing and if he authorizes the transaction then it goes into AVM/SVM workflow for further processing. Post verification, the status of the transaction becomes 'For Auth'.

□         Split File Batch By AVM/SVM: This checkbox is enabled only when Authorization Level is set to ‘Batch Level’ and Batch-Max Instrument Amount is enabled and allows you to split the Batch into multiple PIRs based on AVM/SVM slab depending on the Authorization Type.

□         Send Required: This check box indicates whether you want to route the uploaded transactions through the sending option after authorization 

□         Batch Upload Action: View the reject repair process selected for the product

Standing Instruction-Execution

□         Submit Required, Approval Required, Auto Approval Limit, Verification Required, Send Required: These fields refer to the execution of Standing Instructions. For details, refer to the above section under File Upload  

Validations

□         User Product: This check box allows for a relation between the user and the specified product. If selected, the user can view this product at the time of data entry. By default, this check box is selected  

□         Product Account: This check box allows for a relation between the account and the specified product. If selected, this product and account will be available at the time of data entry. By default, this check box is selected.  

□         User Account: This check box allows for a relation between the user and the account. If selected, the user can view this account at the time of data entry. By default, this check box is selected.

□         Zero Proofing: This check box makes it mandatory for you want to specify the total number of instruments and total amount at the PIR header level for that product.

□         Scrap if not Auth or Send: View the number of days specified after which transactions that are unauthorized or not sent to bank for further processing will be scrapped.  

□         Notify User of Scrap before:  View the number of days specified before which the user will be notified about a definition being scrapped

□         Show Global Attribute for Transaction: If selected Yes, you can allow multi/cross-seller transactions.
