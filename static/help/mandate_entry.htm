Mandate Entry

1.      Enter the relevant values in the Mandate Entry screen to set up a new Mandate. The Mandate Entry screen allows you to set up the personal and contact details of the new Beneficiary under the following heads:

§        Account Details

□         Payer Code: Click the seek icon and select the Payer for whom you want to define the mandate from the 'Payer Help' popup screen displaying a list of drawer codes.

□         Mandate Code: Type a unique code to define the mandate for the selected drawer.

□         Mandate Description: Type the description of the mandate.

□         Mandate Image Description: Enter the description of image if any.

□         Mandate Type: Select the option from the drop down. Available options are Inward Direct Debit and Outward Direct Debit. In case you select Inward direct debit, you need to fill in the beneficiary details tab.

□         Verification Type: Select the appropriate option from the drop down list. The options available for the Bank Admin are Verified, Not Verified, To Be Verified and Rejected. The Corporate User can only view the option- Not Verified.

□         Invocation Type: Select Manual or Automatic as per the requirement.

□         Default: Select the check box if you want to set the Mandate as the default Mandate from a specified date.

□         Channel: This field is defaulted to the source of entry of any transaction.

□         Retry: Check this box if you want to retry the mandate transactions.

□         No. Of Retries: Enter the number (Acceptable between 1 to 9) to set the number of retires.

□         Retry Days: Enter the number of days within which the retry should be done.

Bank Details: These are displayed as per the payer selected above.

§        Mandate Details

□         Time Bound: Select the check box if you want the mandate to be valid for a specific period.

□         Effective From/To: This field is displayed only if you select the Time Bound check box. By default, the current application date is populated in both the fields. Type the start and end date to define the validity period for the mandate.

□         Notify Days before Expiry: Enter the number of days before expiry that you want to set to send the notification.

□         Transaction Limit: Select the check box if you want to define a transaction amount limit for the mandate.

□         Maximum Amount: This field is displayed only if you select the Transaction Limit check box. Specify the maximum amount allowed for the transaction.

□         Transaction Count Limit: Select this check box if you want to limit the transaction based on the count.

□         Frequency: Select the frequency from the drop down. Available options are Daily, weekly or Monthly.

□         Period: Enter the period.

□         Max Occasions: Enter the number of occurrences.

□         Purpose Code:This field provides a seek-list of the appropriate Seller level Purpose Codes maintained by the Seller at the bank. Click to select the purpose code.

□         Purpose Code Description: This field displays the description of the purpose code selected. In case Purpose code is selected as OTHER, then the you can enter the description as required.

§        References

Type any additional information that you want to specify for reference in the following fields:

□        Client Reference

□        Bank Reference

□        Payer Reference

□        Payer Bank Reference

Note: If you are selecting Mandate Type as Inward Direct Debit then Enter the Beneficiary details in the Beneficiary tab.

2.      Click Save to save the details of the Mandate. On saving the entry you are automatically directed back to the Mandate List screen where you can view the newly added Mandate with its status as 'New', indicating that it is awaiting authorization.

3.      Click Back to return to the Mandate List screen without entering a new record.

If you are eligible for authorization, the record will be available in the Mandate Authorization List screen and you can then Authorize the same.

Note:

-         The Effective Date should be subsequent to or equal to the application date. Otherwise, GCP CashWeb displays the error message, “Effective date should not be less than application date.”

-         If the mandate is Time Bound, the effective default date should be between Effective From and Effective To.

-         At BOD when application date is equal to effective date of the mandate, the Mandate Code will be set as the default mandate.
