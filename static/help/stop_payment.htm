About Marking a Stop Pay Request for Instruments initiated outside GCP CashWeb

When a corporate customer issues a cheque through online channel or paper channel using a personal cheque book received during account opening, system does not possess any information related to that instrument. In this scenario, you can stop the payment through the Non CMS Stop Pay screen as explained below. Banks typically charge a fee to stop a transaction on a check already issued to a payee. Stop Payment is done when the instrument is handed over to beneficiary and the Bank / Client is not in possession of the instrument.

Note: If customer marks stop payment from GCP CashWeb Client front end, that request will be tracked as Online Channel. If customer walks into branch and places a request for stop payment, then Bank user on behalf of client will login to GCP CashWeb Admin module and initiate a stop pay request. This will be tracked as Paper Channel.

The Stop Payment center displays all the stop requests along with their authorization states.

You can filter by Date or use the Advanced Filter to retrieve a particular record or a set of records in an account.

Initiating a Stop Pay Request:

1.      Click to open the Stop Pay- New pop-up screen. Type the following details:

§        Reference Number: It is mandatory to define the reference number to help you to query the transaction.

§        Instrument Number: Type the cheque number against which the payment needs to be stopped.

§        Account: Select the debit account number from the Seek list that shows the Client’s DISB_DEBIT accounts for which Cheque Books are issued. Also, these debit accounts should be linked to the client user who will be initiating the stop pay request.

§        Amount: Type the transactional amount of the instrument.

§        Instrument Date: Select the date on which the instrument was issued from the calendar pop up.

2.      Click Save to save the Stop Pay request. System will perform validations and in case there are no errors, the request will be saved successfully.  In case of any errors, system will flash the relevant error on the screen and the same can be edited. The status of the request on Save will appear in the Stop Pay center as Pending Auth indicating that it is available for authorizing.

OR

Click Cancel close the screen without completing the action

Note: Validations:

-         On click of Save, system will validate the cheque numbers provided in the entry screen against those in the inventory tables tagged against "Physical Cheque book" for the client account combination.

-         If available, the request will be saved. If not available, system will display error message "Invalid Cheque Numbers". Click Ok, you are redirected to the 'Stop Payment Request Entry' screen without saving the request.

-         If the system has received the Inward Clearing File and the status of these cheques is Provisional Paid or Paid or Returned, you cannot save the cancellation request. An error message - "Specified cheque numbers are liquidated" will appear on the screen. On clicking Ok, user will be redirected to the 'Stop Payment Request Entry' screen without saving the request.

-         If instrument numbers are available but Inward Clearing File is not received in the system i.e. if the transaction is not available in transaction tables, system will save the request.

Authorizing a Stop Pay Request:

Based on the Maker-Checker concept, if you are eligible for authorization, select the record, validate the correctness of the information provided for the stop pay request and the record. You also have the right to it. The status of the request will change to 'Authorized' or 'Rejected'. Rejected requests will appear in Stop Payment summary screen and will be available for editing. In case you reject the request, it is mandatory to specify the reject reason.

Request can be before Authorizing.

Tip: Click to save the request in .XLS format.

Marking a Cancel / Stop Pay Request for Instruments initiated through GCP CashWeb

Stop payment is an order issued to a bank by clients for not making a payment against an instruction initiated on their account. Stop Payment is done when the instrument is handed over to beneficiary and the Bank / Client is not in possession of the instrument. When the physical instrument is still in possession of the client, the stop pay request is internally referred to as ‘Cancellation’.

After issuance of payment initiation requests, client can subsequently inform the system to stop the payment being made to the beneficiary through the stop payment request. By raising a stop payment request, client can stop the payment instructions for physical as well as electronic instruments.

At any point of time, you, as a maker can view the beneficiary and advice info details that you have entered for a transaction marked as ‘Confidential’. However, while marking stop payment requests for the ‘Confidential’ instructions; all other users can view these details only if the View Confidential flag has been selected for their user IDs in Application Users master

GCP CashWeb allows you to stop or cancel physical as well as electronic payments which are already initiated in the system using the Stop /Cancel Payments feature, at Instrument level.  A stop request can be marked only after the instruction has been authorized by the bank. In case you are assigned Across Entity rights for a corporation, you can stop transactions from processing further for clients spread across different countries.

Note:

-         Only those payment transactions with status as ‘Sent to Bank’ for electronic instruments and 'Received by Bank' in case of physical instruments, are available for Stop payment in GCP CashWeb.

-         Warehoused transactions can be stopped only once they are active in the system.

To mark a stop/cancel payment:

1    In the Payment center, click the Instrument tab to go to the Instrument summary, select the check box of the transaction for which you want to issue the stop payment instruction. These transactions should necessarily show status as Debited.

2    Click More to open the additional actions that can be applied to a transaction and click Stop.

3    Click to process the stop request. The transaction will now be available for authorization.
