About File Upload

This feature enables you to upload multiple Collection transactions in the system. This reduces the efforts required for manually entering the bulk transactions through data entry screen. On uploading the file, GCP CashWeb carries out file format validations followed by data level and field level validations. Once the transactions are successfully uploaded, they are automatically closed and appear in authorization screen. Auto authorization and auto release depends upon the parameters set in My Product master .

Imp: All transactions initiated in GCP CashWeb have to be authorized and sent to bank for further processing within a time frame set in the My Product master.  At both these stages, i.e. when the status of the transaction is For Auth- the Client Checker and For Send- the Client, will receive a notification of the pending action.

In case a transaction remains unauthorized, system will notify the client before scrapping the transaction based on the ‘number’ of days entered in the ‘Notify User of Scrap before’ field in thein the MyProduct master.

Based on the ‘Scrap if not Auth or Send’ limit set in the MyProduct master by the Maker, system will scrap all such transactions with status- ‘For Auth’ and ‘For Send’. Notification of the same will be sent to the Client and the Client Checker (authorizer) of the same. This is applicable for all the Client initiated transactions.

The File Upload center allows you to view the status as well as the details of the uploaded file. The display is refreshed every few minutes, to provide you with the most updated information.

□         File Name: This field displays the unique file name without the extension

□         Date and Time: This field displays the date on which the multiple payment file is uploaded.

□         Status: This field displays the status of the uploaded file. The different states of upload can be: New, In Queue, Running, Aborted, and Completed.

□         Remarks: This field displays any remarks that may have been generated by the system, indicating the state of upload. In case, any errors occur during the upload process, relevant error messages will be displayed to you in this field. This will help you to identify the problem with the uploaded file as shown below:

Uploading Multiple Transactions:

1.      Click to open a pop-up screen that allows you to upload the file. Enter the following details:

§        File Format Type: Select the format type from the drop down list of available formats. These formats are unique across clients. If a map code is defined for a client, it cannot be reused. During file import in GCP CashWeb, the Map Code field displays the client specific map codes defined in this master

Note: Each client can have different and multiple formats which can be tagged to a Map Code.  These Map Codes, once assigned to a client, can be selected by a client to upload respective formats.

§        File Name: Select the file from your network, using the Browse option. If you are entering the name of the file, ensure that you are specifying the absolute path of the file.

2.      Click Upload to upload the selected file. GCP CashWeb will run file level validations as well as data level validations on the file. In the course of validating an uploaded file, if system encounters any errors that can be repaired, the uploaded file is moved to 'Reject Repair'. If the errors cannot be repaired in ‘Reject Repair’, then the system aborts the filer. This triggers an event and a notification of the reject reason is sent to the Client as well as the Bank User by E-mail or SMS as specified in the Event Notification master. The reject report is available for viewing or downloading in the File Upload summary screen. If both field and data level validations are successful, system uploads the file successfully and displays the message “File Uploaded Successfully”.

OR

Click Cancel to cancel the upload request.

IMP:

-         The maximum file size allowed for upload is 50MB.

-         In the course of validating an uploaded file, if system encounters any file related errors, the same is aborted. In case of data level errors, the file is sent for Reject Repair. This triggers an event and a notification is sent to the Client as well as the Bank User by email or SMS as specified in the Event Notification master. The same can also be viewed on-screen in the File Upload summary screen. You can also view or download the Reject report. The rejected file appears in the Payment center with status -For Repair and  user can repair the errors and validate the file for further processing.

3.      Once the transaction is uploaded successfully, it appears in the Collection Center with status as Draft with the option to Discard it. Here on the same workflow steps (Authorization and Send), as in the case of Manual Data Entry have to be followed. If Auto Auth and Auto Send functionality is enabled then transaction will directly appear in ‘Sent to Bank’ status.

Note:

-         For automatically closing the uploaded transactions, you have to set the Submit Required flag as unchecked in the My Product master.

-         In case of auto authorization and auto release, you need to set the Approval Required & Send Required flags unchecked in the My Product master.

-         The action on the uploaded file depends on the option selected for the File Upload Action field in the Collections My Product Master Entry >> Workflow >> File Upload.

Viewing the Reject Report:

In case an uploaded file gets aborted due to file level validation failure or appears in Reject Repair due to data level validation failure, you can download the Reject Report to understand why the file was aborted or rejected.

1    In the File Upload screen, click the Download hyper link of a file under the 'Reject Report' column  which is either aborted or rejected.

2    In the File Download popup that appears, click Open to view the generated report or you can Save it on your local machine in .pdf format.  

Repairing Rejected Transactions

When you upload a file with multiple transactions, GCP CashWeb performs all the validations that are performed at the time of data entry. If the system encounters any errors while uploading the file, you can edit or repair the errors at instrument level and send it for further processing.

1    In the Collection Center, click the Instrument tab to open the instrument summary screen.

2    From the Status drop down, select Pending Repair to retrieve all the instruments that are available for repair

3    To repair an instrument, click  Edit Instrument . The Instrument Edit screen opens displaying the error reason in the header area.

4    Make the necessary changes. System refreshes the screen with the updated value and the error displayed on the screen disappears.

5    Click Update to save the changes, and then Back to go to the Collection center.

6    The status of the repaired transaction changes to ‘For Submit’ and the Submit button gets enabled.

7    Click Submit and the status of the transaction changes to ‘For Auth’.

8    To Authorize a repaired transaction, click Authorizing a Collection Transaction and Releasing it to the Bank

□         Payer Country is Mandatory

□         Credit Account and Payer Account cannot be same

□         Details of Collections value should contain A-Z or a-z or 0-9 Only

□         Invalid Collection  Amount

□         Details of Collection is Mandatory

□         Client does not have setup for the Collection Currency

□         If the file contains any of the blacklisted characters and Oracle SQL keywords (like Insert/Update/Union etc) file will be outright rejected.

□         Mandatory advice info is missing

□         Mismatch in the advice info sequence

□         Information is not in proper format
