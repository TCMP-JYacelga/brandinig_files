User Master Entry

1.      In the User Master Entry screen complete the user details.

□         Category: Select the Category that you want to assign to the user. Category Code of every category that is authorized is available in the drop-down box for selection.

□         Client: Click the Seek List to select the Client to which the User belongs.

□         User Code/ ID: Click the Seek List to open the pop up displaying the User Code, User Name and the Description as defined in the system by the Bank Admin. Select the User Code of the User that you want to assign to the Category.  

□         User Name and Description: These fields are auto-populated on selecting the User Code with the values existing in the system .

□         Language: Select the language for the user. All the languages that are supported by the system are available for selection. The next time the user logs in, the screens of the application will display information in the language selected.

□         E-mail: This field is auto-populated on selecting the User Code with the values existing in the system. The login password will be sent to the user on this mail id.

□         Mobile Number: Type the mobile number of the new application user.

□         Fax Number: Type the fax number of the new application user.

□         Across Client: A user may belong to a client, which is a sub entity of a large Corporation. The Across Client option allows a particular user to work across other clients that fall under the same Corporation. Check this box if you want the user to work across clients.

□         View Confidential: Select the check box if you want to allow the user to view the beneficiary and advice information of payment requests, which are marked as confidential. Moreover, this selection also indicates that while marking the authorization for confidential payment requests, the user can view the beneficiary and advice info details in the relevant authorization screens. At any point of time you can change the value set for the View Confidential flag

□         Event Template Code: Select the appropriate Event Template Id to be assigned from the seek list available.

□         Template Only User: Check this box to allow the user to create transactions from templates only.

2.      The Detail area displays a menu wise list of the application functionality. Depending on the category that you have selected, the rights that have been assigned to that category are automatically ticked. However, you can assign a user lesser rights than those of the parent category. Change the default access rights by clearing the Rights check boxes against the respective functionality.

3.      Assign the Types of Rights to the user.

Select the rights depending on the role the defined user is playing for the client.

□         View: Selecting this check box assigns viewing rights

E.g.: If you select the View check box against the function Beneficiaries under Masters, the users in the category you are defining can view all the records created in the Beneficiary master.

□         Edit: Selecting this check box assigns editing rights to that category

Note: To assign the edit rights for a category, it is mandatory to select both the View as well as the Edit rights.

E.g.: If you want to assign the right to edit the product records, against the User Product function under Masters, select both the View and the Edit check boxes. Doing so implies that, the users in the category you are defining can create as well as edit the existing records in the Product master.

□         Auth: Selecting this check box assigns authorizing rights.

Note: Since you cannot edit the Authorization Screens, the Edit option is not available for these screens.

4.      Click Save to save the new user. On Save, the new user is added to the User List, with its state specified as New. This implies that the new addition has not yet been authorized.

Or

Click Back to exit from this screen without saving, and return to the User List screen.
