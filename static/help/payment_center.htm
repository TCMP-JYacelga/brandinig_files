About Payment Transactions

A manual payment request is initiated through data entry by the client to the System Bank branch to pay the beneficiary a particular sum of money. This request provides the System Bank with the necessary details such as the client’s account, the details of the beneficiary bank and amount to be paid. The menu bar shows the filter that can be applied to retrieve a transaction satisfying the selected criteria, along with the Actions that can be taken on a transaction.  

The below listed actions can be performed on transactions:

□         New: Click New to initiate a new transaction or template.

□         Submit: This action is available after a transaction is successfully saved. Click Submit to submit the transaction for verification or authorization.

□         Verify: This action is available after a transaction is submitted; It is based on the Payment MyProduct setup and the rights available to the user. Click Verify to verify the transaction details.

□         Authorize: This action is available on Submit/Verify and the authorization level set in the Payment MyProduct setup. Click Authorize to authorize a transaction.

□         Second Auth: This action is available to the users who are assigned the rights for second authorization. Transactions that would require this second authorization are as below:

–Transaction above the threshold limit of single authorization.

– Transactions which are initially authorized by the user having rights within the threshold limit.

– This method is applicable for manual entry as well as file upload.

□         Send: This action is available after a transaction is successfully authorized. Click Send to send the transaction to the bank for further processing.

The below options are listed under More and are related to the above actions:

□         Reject: This option is available to the authorizer to Reject a transaction.

□         Hold: This option is available for a transaction that is authorized but not sent to the bank. Click Hold to hold back the transaction for any particular reason or pending action

□         Release: This option is available only for transactions put on hold. Click Release to release the transaction

□         Stop: This option is available for transactions that are authorized and the status is Pending Release. Click Stop to stop the payment transaction.

□         Discard: This option is available for all transactions with status as Draft and Pending Submit. Click Discard to delete the transaction.

Note: All the discarded transactions can be viewed in the summary screen with the status as Deleted and are available for cloning. Once replicated, they are available in the  for initiating the transaction.

□         Download: This option is available for downloading transaction details

□         Print: This option is available for Printing transaction details.

Initiating a Manual Payment Transaction:

1.      Click the icon in the menu bar. The New Payments pop up shows a list of all the MyProducts available for the client.

§        The Transaction radio button is selected by default and allows you to initiate a new Payment Transaction.

§        Select New From Template to create a transaction using a transaction template. You can also initiate a transaction by cloning a Template with status as ‘Template Ready’.

Ref:

-         For more details on creating a Transaction Template, refer to ‘Creating a Transaction Template’.

-         For initiating a transaction, refer to Initiating a Transaction using a Template.

2.      Select a MyProduct that you want to use for processing the current transaction.

The MyProducts are grouped under two heads- My Products and Bank Products.

Group of bank products clubbed together are displayed under MyProducts and are usually associated with -

□        Salary

□        Vendor Payments

□        Reimbursements

MyProduct having only one bank product are displayed under Bank Products like-

□        Book transfers

□        Hi-Value Payments

3.      Entering Batch Details: For Multiple Instrument Transaction- Batch Pay, you will be directed to the Batch Header screen.

4.      Entering Instrument Details: For Single Instrument Transaction- Quick Pay ,you will bypass the Batch Header and will be directed to the Instrument Entry/Quick Pay Entry screen.

5.      Submitting a Transaction:  On successful initiation, the transaction will appear in this screen with the status showing - Pending Submit. Select the record and click the  icon to make it available for authorization. If successfully submitted, system displays a message confirming it and notifies the Client Checker (authorizer) that the transaction has been entered and closed and is awaiting authorization.

§        In case Authorization Level field in My Product master is set to Batch level, you can Submit, Authorize and Send the transaction at Batch level.

§        In case Authorization Level field in My Product master is set to Instrument level, you can Submit, Authorize and Send the transaction at Instrument level.

§        In case of Quick Pay or where the Batch has only one instrument, actions such as Submit, Authorize and Send will always be at Instrument level.

Note: In case you are assigned Across Seller rights for a corporation, you can Submit, Verify, Authorize, Reject, Send, Stop, Hold and Release and Send transactions of clients spread across different countries. This feature is based on the settings maintained for the User- Client relationship in the Seller Corporation master at bank end.

6.      Based on the Maker-Checker concept, if you are eligible for authorization, the status of the record will show - Pending My Auth, Pending Auth or Pending Verification (if Verification Required flag is selected in the Payment MyProduct master)

Editing a Payment Transaction:

1.      Click Edit to go to the Batch Header screen in case you are at Batch level, or the Instrument Edit screen if you are at Instrument level, where you can edit a previously initiated transaction.

2.      Click Update to save the changes.

Note:  You can modify a transaction only till such time that it is not Submitted.

Viewing a Payment Transaction:

1.      Click View to go to the Batch Header screen in case you are at Batch level, or the Instrument View screen if you are at Instrument level,where you can view a previously initiated transaction. You cannot modify the transaction in this screen.

Note: While viewing the details for Instrument type Cheque, DD and Cashier Order if encashment is done by the bank then the instrument details shows the indicator Instrument Encashed and its respective remarks in Encashment Remarks if entered by the bank user.

2.      Click Back to return to the Payments center.
