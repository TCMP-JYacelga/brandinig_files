Payer Authorization List

Collection >> Masters >> Payers Auth

GCP CashWeb follows the Maker-Checker concept, which ensures that every new Payer Entry, or any additions or changes made to an existing entry by the Maker are authorized by a Checker. Similarly, all requests such as enabling, disabling or  rejecting records also need to be authorized in order to fulfil them. A Checker verifies the authenticity of these records before authorizing them.

The Payer Authorization List tells you at a glance whether a particular entry has been authorized or not. This is indicated in the Status column, which displays the current state of the corresponding entry.

Authorizing a Payer:

§        As Checker, select the Accept check-box to authorize the request.

§        You may also select the Reject check box to reject the request. While rejecting a request, it is mandatory to type the reason for rejection. Type the reject reason and click OK.

§        Depending on whether the entry has been authorized or not, the Status column takes different values.

Status Requiring Authorization

Pre- Authorization State

Post- Authorization State

New Definition: Any new definition that has been created

New

  • If Authorized: Enabled

  • If Rejected: New Reject

In this case, the definition can either be modified or deleted.

Modify Definition: Any existing definition that has been modified

New or Enabled

  • If authorized: Enabled

  • If Rejected: Modified Reject

(a) If the modified transaction was the New one, it can either be modified or deleted.

(b) If the modified transaction was the Enabled one, the Modify Request has to be discarded so as the definition retains the Enabled state.

Disable Definition: Any enabled definition that has been disabled by the Maker

Disable Request

  • If authorized: Disabled

  • If Rejected: Disable-Req Reject

In this case, the Disable Request by the Maker has to be discarded so as the definition retains the Enabled state.

Enable Definition: Any disabled definition that has been enabled by the Maker

Enable Request

  • If authorized: Enabled

  • If Rejected: Enable-Req Reject

In this case, the Enable Request by the Maker has to be discarded so as the definition retains the Disabled state.

 

§        A successfully authorized record will also appear in the Payer List screen with the status - 'Authorized' and you can 'Enable' or 'Disable' it provided you have the rights to do so.

Viewing a Payer Definition:

1.      Click View to go to the Payer Master View screen where you can view a previously defined master. You cannot modify the definition in this screen.

2.      Click Back to return to the Payer List screen.
