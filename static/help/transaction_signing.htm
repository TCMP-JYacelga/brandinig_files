The detailed workflow of Transaction Signing using Smart Card is as follows:

(1) Customer User logs into GCP CashWeb application via CAAS

(2) User selects to view the transaction available to be authorize on the Batch / Instrument Authorization screen, based on the authorization level (PIR / instrument)

(3) User selects the individual transaction and clicks on authorization button

(4) If the transaction selected has Taiwan domiciled account, User is prompted with option to choose between token or smart card signing as below:

(5) If User selects token option, the transaction is authorized based on the current signing logic

(6) If User selects the smart card based option,  signing data HMAC of the attributes displayed on the screen is generated using the same logic as used for transaction signing.

The following screen appears for batch authorization. The sign button is enabled when the User has selected the Smart Card option.

On clicking Sign, the following screen appears and User has to enter the appropriate passwords to authorize the batch/transaction.
