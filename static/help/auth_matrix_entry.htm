Authorization Matrix Entry

The Authorization Matrix Entry screen allows you to set limits for each product or account. For each limit, you need to the assign the users who are required to approve the transaction.

Defining an Authorization Matrix:

1.      Enter the following details

§        Product: Select the appropriate product from the drop down list

§        Authorization Type: Displays the category of authorization such as Product or Account as defined for the authorization matrix. In case of Account, you need to select the account number.

§        Authorization Parameter: This field is non editable. This field is automatically populated with the value specified in the “authorization Matrix List” screen.

§        Currency: Click and select the currency. Authorization matrix can be based on the currency factor also. Click here to learn more about currency based authorization matrix. This field is defaulted to seller base currency.

§        Limit From and Limit To: Type the lower and the upper transaction limit in the respective fields. This limit is a range of transaction amount that decides the application of the authorization rule. Any transaction initiated will be saved only if it satisfies the specified limit.

2.      Click Save to save the definition.

3.      Click Add Detail to go to the Authorization Matrix Detail Master to add the following:

§        Level No.: Type the sequence number for the level that you are defining. If the authorization matrix definition that you are setting up has multiple levels, the authorization process will proceed as per the sequence number specified here.

Note:

-         An authorization matrix definition may include as many levels as required, depending on your organizational or functional complexity.

-         In case of multiple levels within an authorization matrix, the next level of authorization in sequence can begin only after the previous level of authorization has been completed.

§        No. of Users: Type the number of users that will be involved in authorization process at this level.

4.      Select the User Groups for that level from the list below.

5.      Click Save to save the new details. The Add Detail button is again enabled in case you want to add more levels and users. Repeat the process for every new level and user to be added.

The level definition is added to the grid which displays all the levels that you have defined for the authorization matrix definition, along with the users and user groups. Against every level definition is the option to edit, view and delete.

Note: Click Edit to go to the Authorization Matrix Detail Master where you can edit the details.

6.      Click Update on completion of the level definition to go back to the Authorization Matrix List screen

The authorization matrix definition is added to the list in the Authorization Matrix List screen with its status as New, where it awaits authorization.
