About User Subscription Master

Subscriptions allow you to identify the event code and recipient combination for various events triggered in the system along with the delivery mode through which the event notification is sent to the recipient. This setup is done by the bank based on the preloads provided in the system.

The User Subscription Master allows you to assign a subset of Subscriptions from the Customer User Category template to the various Users of the Client.

IMP: Ensure that you select the client for whose user you want to assign Subscriptions using Switch Client.

To assign Subscriptions to a User:

1.      In the User Subscription List screen, select the User to whom the Subscription is to be assigned from the drop down list that displays all the Users of the Client.

Note: You may specify the filter criteria and click Filter to retrieve a particular definition.

2.      Click Add Subscription to open the User Subscription Entry screen. This screen displays the Subscriptions that are available for the user.

Note:

-          A newly assigned User Subscription that is yet to be authorized can be discarded.

-          In case a Subscription is already assigned to the User, the screen will display an error with the relevant message- ”Record already exists”.

-          In case you select a new  user, repeat the whole process to  assign a Subscription.

To enable the User Client Rights definition, it is mandatory to authorize the requests based on the Maker-Checker concept. The checker can view the new definitions in the User Subscription Authorization List screen and authorize the same provided the rights to authorize are assigned to that checker.

On successful authorization, the User Subscription can be viewed in the User Subscription List screen with the status as Authorized.

Editing a Subscription:

1.      Click Edit to go to the User Subscription Entry screen where you can edit a previously defined master.

2.      Click Update to save the changes.

Note:

 You can modify the definitions if they are in any of the following states:

-         New : Any new definition is not yet authorized and is in the new state.

-         Modified : Any definition that you have modified and not yet authorized and is in the Modified state.

-         Authorized: The definition that is authorized and is in the Enabled state.

The edit option is not available for the definitions that have the state as Disabled, Disable Request, Enable Request, Disable Request Rejected, and Enable Request Rejected, Submitted, Authorized.

On saving the modifications, the definition state changes to Modified. To enable the definition with the Modified state, the definition has to be authorized.

Viewing a Subscription:

1.      Click View to go to the User Subscription View screen where you can view a previously defined master. You cannot modify the definition in this screen.

2.      Click Back to return to the User Subscription List screen.
