Category Master Entry

The Category Master Entry screen displays a menu wise list of the application functionality along with the type of rights that can be attached to that category. It also allows you to define a new category as shown below:

1.      Define a new Category to which the client user will belong.

Type the following Category details:

□         Category Code: Type a code for the category. The category code is a unique value used to reference that category throughout the system. The category code can contain a maximum of ten characters.

□         Description:Type a brief description for each category.

Note: For both the fields, Category and Description, the system does not support the following special characters: “ \ ~ ^ and ‘.

2.      Assign rights to the category that are listed under Type of Rights depending on the role the defined user is playing for the client.

Select the rights depending on the role the defined user is playing for the client.

□         View: Selecting this check box assigns viewing rights

E.g.:If you select the View check box against the function Beneficiaries under Masters, the users in the category you are defining can view all the records created in the Beneficiary master.

□         Edit: Selecting this check box assigns editing rights to that category

Note:To assign the edit rights for a category, it is mandatory to select both the View as well as the Edit rights.

E.g.: If you want to assign the right to edit the product records, against the User Product function under Masters, select both the View and the Edit check boxes. Doing so implies that, the users in the category you are defining can create as well as edit the existing records in the Product master.

□         Auth: Selecting this check box assigns authorizing rights.

Note: Since you cannot edit the Authorization Screens, the Edit option is not available for these screens.

3.      Click Save to save the new category entry. On Save, the new category is added to the Category List , with its state specified as New. This implies that the new addition is yet to be authorized.

Or

Click Back to exit from this screen without saving, and return to the 'Category List' screen.
