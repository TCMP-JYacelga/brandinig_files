About Cheque Book request

GCP CashWeb provides the facility to their registered customers to initiate a Cheque Book request through online channel or through paper request. To support this new functionality, GCP CashWeb provides a frontend for Bank’s clients to initiate a cheque book request through online channel. In case of paper request, Bank Admin can login to GCP CashWeb Admin console and initiate a cheque book request on behalf of its client.

The Cheque Book Requests summary screen displays all the cheque book requests along with their status. You can delete the records whose status shows as ‘New’.

You can filter by Date or use the Advanced Filter to retrieve a particular request based on the details specified..

To initiate a new Cheque Book request:

1.      Click the New icon to open the Cheque Book Request Entry screen which allows you to initiate the cheque book request. Enter the following details:

  1. □         Cheque Book Reference No: Specify the unique reference number for the new request. This will help the user to query his request to check the status.

    □         Account Number: Select the debit account from the seek list on which the client user wants to have a cheque book.

Note: The Account Number seek list will display all the  DISB_DEBIT accounts of the selected client for which Cheque Book Facility flag is set to ‘Yes’ in the Client Accounts and the Client Additional Accounts masters. Additionally, these debit accounts are linked to the client initiating the cheque book request

  1. □         Number of Leaves: This indicates the number of cheques in a booklet and is set to 50

    □         Number of Booklets: Select the number of cheque books required for the specified debit account.  Options are 1-10.

    -         Delivery Mode: Select the mode of delivery for the cheque books i.e. whether client will pick it up from the branch specified in the request or Bank has to courier it to the address specified in the request. Options available are ‘Collect from Branch’ & ‘Mail to Address’ and is defaulted to ‘Collect from Branch’.

    -         Mailing Address:

In case you select 'Mail to Address', specify your detailed address along with Country, State, City and Post Code in the fields that follow.

Note:

-         By default this field displays the mailing address of the client from the Client Master maintained at backend.

-         5 address lines – Free text field with maximum length as 30 Char each

-         Country –Seek list from Country master

-         State –Seek list from State master

-         City –Seek list from Clearing location master

-         Post code–Free text field with maximum length of 6 Char

□         The customer can modify this address. Bank will then courier the cheque books on the address specified in the request.

□         If the mapped client address fields contain a non-English address, then the address field on the screen will be populated with the non-English address.

In case you select 'Collect from Branch', specify the detailed address of the Branch along with Country, State, City and Post Code in the fields that follow. along with the below fields:

  1. □         Collection Branch: This field is available only if Delivery Mode is selected as ‘Collect from Branch’. Select the system branch from where client will pick up the cheque books. The Seek will display all the system branches of the client whose user is logged in.

    Authorized Person Name: Type the name of authorized person. Authorized person is the person designated by Client to collect the cheque book.

    Authorized Person ID Type: Select the identification proof for authorized person. The available options are: Driving License, Employee Card, National ID Card and Passport.

    Authorized Person ID Number: Type the authorized person ID number depending upon the ID type.

Note: The above four fields- Collection Branch, Authorizes Person Name, Authorized Person ID Type and Authorized Person ID Number are visible on the screen and mandatory only when ‘Delivery Mode’ is selected as Collect From Branch.

2.      Click Save to save the request. If successfully saved, this record will appear in the ‘Cheque Book Requests’ summary screen  for authorization with status as New or click Back to return to the 'Cheque Book Request' summary screen without initiating a request.

Note:

-         Every request has to be authorized in order to process it further. As the authorizer, you need to validate the correctness of the information provided for the Cheque Book Request

-         Authorization is based on the Maker Checker concept and you can authorize a request as Checker only if you have been assigned the rights to do the same.

To authorize a request:

Select the record, validate the correctness of the information provided for the cheque book request and click the Authorize icon to authorize the record. You also have the right to Rejector Discardthe request. The request status will either be 'Authorized' or 'Rejected'. Rejected requests will appear in Cheque Book Requests summary screen for editing. In case you reject the request, you have to mention the reject reason. Successfully authorized request will be sent for further processing.

Note: Once the Bank receives the data from backend for printing the cheque books, the Bank user will generate a report on demand or schedule a report of the same. This report will be sent to the printer vendor for printing the cheque books. Once the cheque books are printed and in custody of the Bank, the same will be delivered to the client.

Editing a Stop Pay Request:

1.      Click Edit to open the Stop Pay Edit pop-up where you can edit a previously initiated Stop Pay request.

2.      Click Update to save the changes.

Note:  You can modify a transaction only till such time that it is not Submitted.
