My Product Advice Info.

1.      Click the Add button in the 'MyProduct Advice Info. List' screen to open the MyProduct Advice Info. screen. This screen allows you to define a maximum of 25 MyProduct Advice Information fields.

2.      To add a new Instrument level advice information field, type the following details:

§        My Product: Select the product from the drop down list showing all the existing MyProducts defined for the client, for which the client requires the Instrument level advice information to be defined. Only those products defined for the given client in the client product screen are visible.

§        Advice Info. Code: Type a code that will be assigned to this advice information field and that will be its unique identity throughout the system.

§        Advice Info. Type: This field will be defaulted to Single Set. With this option user can specify Single invoice details against single transaction. The other option is Multi Set.

Note: You can define either multi-set or single set advice information fields for a MyProduct. System will not allow defining both multi-set and single set fields for the same MyProduct.

§        Label Description: Type a description of the advice info code/field defined above.

§        Mandatory Flag: Select Yes if this advice info field is mandatory for the instrument. If you select Yes, then while entering transactions you will not be able to save a transaction under this client-product combination unless you have entered the advice information details.

§        Data Type: Select the data type for this Instrument advice information field. The options available for selection are String, Numeric, Date and Amount.

Note: You can define maximum 3 date fields, 4 alphanumeric fields, 5 numeric fields and 6 amount fields of informations for each product.

§        Length: The values in this field are auto-populated depending on the selection in the Data Type field. In case you select String, Numeric or Amount this field shows the maximum length that the information value can take. In case of Date, it will display the format for entering the date. You are also given the options to choose from the drop-down list provided for Date.

§        Min Value: This field is displayed only if you have selected the Data Type as Amount. If you select Yes, you need to type the minimum amount that the information value can accept.

§        Max Value: This field is displayed only if you have selected the Data Type as Amount. If you select Yes, you need to type the maximum amount that the informtaiont value can accept.

§        Validation Flag: This field is enabled only if you have selected the Data Type as String. If you select Yes here, you need to enter the validation rules.

3.      Click Save, in case Validation Flag is selected as Yes, the Add Detail button is enabled and you can add validation rules.

4.      Click Add Detail to open the My Product Advice Info. Detail screen. Enter the following fields:

§        Value List Nmbr: The Value List Number is auto-populated serially with each new rule added and specifies the sequence in which the advice information fields will appear for entry during transaction initiation.

§        Advice Info. Value: Enter the length of the alpha numeric value for that rule.

5.      Click Save to go back to the ‘My Product Advice Info.’ screen. The newly added rule can be viewed in the details area with the option to delete the same.

Note: To add one more rule, repeat steps 2 and 3.

6.      Click Update once you have defined all the validation rules to go back to the ‘My Products Advice Info. List’ screen.
