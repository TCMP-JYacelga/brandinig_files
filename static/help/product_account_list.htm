About Product Account Definition

After you define your products in the MyProducts master, you need to set up a relationship between the products and the accounts as an added security measure. Such a relationship ensures that while initiating a transaction for a product, only the accounts that have been assigned to the product will be available. This ensures that transactions for this product are initiated only through the assigned accounts. Setting Up a Product Account Definition

IMP: Ensure that you select the client for which you want to set up the  definition using Switch Client.

Assigning Accounts to a Product:

1.      Specify the following details:

§        Product: Select a product that you want to associate with the client account. The list displays all products assigned to the client in the MyProducts master.

The below filters are not mandatory.

□         Account: Select the account that you want to assign to the product

□         Account Description: Select the brief description of the product.

□         Currency: Select the currency in which the account is held.

2.      Click the Filter button to retrieve the definition.

3.      Select the Assign check box of the accounts from the list below, to associate those accounts to the Product.

The status for those accounts changes from Not Assigned to New. This indicates that a relationship is set between the Product and the accounts and it awaits authorization. Against every such relation are options to enable, disable or discard that user account definition.

Note:  In case you select a new Product, repeat the whole process to assign accounts to that Product.

To enable the Product Account definition, it is mandatory to authorize the requests based on the Maker-Checker concept. The checker can view the new definitions in the User Account Authorization List screen and authorize the same provided the rights to authorize are assigned to that checker.

On successful authorization, the Product Account definition can be viewed in the Product Account List screen with the status as Authorized.
