Authorizing a Standing Instruction

IMP:

-         All transactions initiated in GCP CashWeb have to be authorized and sent to bank for further processing within a time frame set in the My Product master.  At both these stages, the Client Checker  (when the status of the transaction is Pending My Auth) and the Client, (when the status of the transaction is Pending Release) will receive a notification of the pending action.

-         In case a transaction remains unauthorized, system will notify the client before scrapping the transaction based on the ‘number’ of days entered in the ‘Notify User of Scrap before’ field in the MyProduct master.

-         Based on the ‘Scrap if not Auth or Send’ limit set in the MyProduct master by the Maker, all transactions with status- ‘Pending Auth’ and ‘Pending My Auth’, exceeding the limit will be scrapped bt the system and notification of the same will be sent to the Client and the Client Checker (authorizer). This is applicable for all the GCP CashWeb Client initiated transactions.

Authorizing a Standing Instruction:

Note: You can authorize an instruction only if you have been assigned the rights for authorization.

Authorizing Batch/ Instrument by Client

Authorizing a Batch/ Instrument by Token Device:

1.      In the Standing Instruction summary screen, select the SI with status showing Pending My Auth and click to open Select Signature Type popup.

2.      Select Token option and click Continue to go to Batch / Instrument Authorization Summary screen.

3.      The Token Entry Field displays a unique system generated number.

4.      Enter the number generated in the ‘Token Entry Field’ into the Token Device and press the Generate to generate a unique numeric code. This is the secure Digital Signature.

5.      Enter the unique numeric code in the Digital Signature field for the selected transactions.

6.      Click Sign & Submit to complete the authorization process.

 

Authorizing a Batch/ Instrument by Smart Card:

1.      In the Standing Instruction summary screen, select the SI with status showing Pending My Auth and click to open Select Signature Type popup.

2.      Select Smart Card option and click Continue to go to Batch / Instrument Authorization Summary screen.

3.      In the Batch/ Instrument Authorization screen, the Sign button is enabled when you select the Smart Card option.

4.      Click Sign. The e-Signer box appears.

5.      Enter the appropriate password to authorize the batch/transaction.

§        On successful authorization the screen will display a message confirming the same and the status of the record changes to - Authorized.

§        To reject or delete the transaction, select the transaction and click Reject or Discard. To reject a transaction, it is mandatory to specify a reject reason.

Note: All the discarded transactions can be viewed in the summary screen with the status as Deleted and are available for cloning. Once replicated, they are available  for initiating the transaction

§        You also have the option to enable or disable a Standing Instruction transaction. Click More to select Enable or Disable.

§        For Instrument Authorization, follow the same process as explained above.

Note: Once an SI transaction is authorized, the process ends; On the Beginning of Day (BOD) of the effective date set in the SI, a new transaction will be generated based on the values defined in the SI and there on it will be processed as a regular transaction.

Second Level Authorization

Depending upon the settings done in the bank GCP systems, if the payment amount is above a certain threshold, a second bank user is required to view the transaction and authorize the same in addition to the existing authorization workflow,

§        Second Auth: This action button is available to the users who are assigned the rights for second authorization. Conditions that would transfer the transaction to second level authorization are:

–Transaction above the threshold limit of single authorization.

– Transactions which are initially authorized by the user having rights within the threshold limit.

This method is applicable for manual entry as well as file upload.
