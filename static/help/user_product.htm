User Product List

Admin >> User Setup >> User Products

For a user to be able to successfully initiate, edit, or add transactions related to a particular product, there must be a valid User Product, wherein the User and the Product have been assigned a relationship. Such relations are set up in the User Product List. With this feature, users can initiate transactions for only those products that are assigned to them. The User Product screen also displays the existing user product relationships along with the client, description and the present status.

IMP: Ensure that you select the client for which you want to set up the  definition using Switch Client.

To Assign Products to a User:

1.      Specify the following filter details:

§        User: Select the user to which you want to assign a product.

§        The below two filters are not mandatory:

§        Product Description: Select the unique identifier for the product as you have defined in the My Products master.

§        Client: Select the name of the client with which you have logged into GCP CashWeb.

2.      Click the Filter button to retrieve the definition.

3.      Select the Assign check box of the products from the list below, to associate those products to the user. The status for those accounts changes from Not Assigned to New.

This indicates that a relationship is set between the user and the products and it awaits authorization. Against every such relation are options to enable, disable or discard that user account definition.

Note:  In case you select a new  user, repeat the whole process to assign products to that user.

4.      To enable the User Product definition, it is mandatory to authorize the requests based on the Maker-Checker concept.

5.      The checker can view the new definitions in the User Product Authorization List screen and authorize the same provided the rights to authorize are assigned to that checker.

On successful authorization, the User Product definition can be viewed in the User Product List screen with the status as Authorized.

Other Possible Tasks

IMP:

-         If you disable a Product definition that you have set up in the MyProducts master, all the corresponding User Product definitions that you have set up for this product are automatically disabled.

-         When you enable the Product definition, the corresponding User Product definitions will not be enabled automatically. You have to enable each of them manually.
