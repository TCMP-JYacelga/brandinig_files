GCP CashWeb                                       

 

Instrument Entry/ Quick Pay

1.      The Instrument Entry screen allows you to enter instrument-level transaction details.

To initiate an instrument specify the following details:

Product Details

□         Payment Currency: Select the Payment currency from the drop down list that displays all the currencies pertaining to the My Product/UDE Product selected. It is mandatory to select the Payment Currency before selecting the Payment Product.  

□         Payment Product: Select the payment product for the specified MyProduct transaction from the drop down list. In case you do not select a product, the system derives the Payment Product based on the transaction values entered. GCP CashWeb displays the mandatory fields depending on the selection made for the Delivery Mode field.

Depending on the Payment Product selected by you, the ”Instrument Edit” screen displays the relevant fields associated with the selected product. Therefore, the entry screen fields may differ with each product (e.g. Internal Funds Transfer, Dividend Warrant, Demand Draft, Customer Cheque, Cashier Order, Automated Clearing House, Telegraphic Transfer, Local Wire and Giro).

In case you do not specify the Payment Product, GCP CashWeb displays the mandatory fields depending on the selection made for the Delivery Mode field.

□         Client Reference:  Type a unique reference code to identify the client.

□         Record Reference: For Quick Pay the text field is defaulted to blank. For Batch Payments, the text field is defaulted with the Batch Reference from Batch level and is editable. You can edit the reference number. This field is either mandatory or optional based on the parameters set by the bank. These settings ensure that duplication of the record reference number is avoided and the same can also be blocked for a specified number of days.

A reference number to uniquely identify the Payment Initiation Request (PIR)entry. This reference number will help you while querying and retrieving the required PIR.

□         Debit Account: Select the debit account number from the drop down list. All accounts attached to a client except for those maintained as WASH are displayed in the drop down. On selection of the account number, system will populate the account details such as account title, account currency, available balance and available limit. This field is in read-only mode if payment product is at Batch Level and editable if Debit Account is at Instrument level. The Account currency is displayed along with the Debit Account.

Beneficiary Details

□         Beneficiary Code: Once you select the beneficiary the related details will be displayed in the account number and account branch fields.

□         Name: This field automatically populates the beneficiary name that you have specified in the Beneficiaries master for the selected Beneficiary. This field is non editable

More Details

Click the More Details tab to open the 'Registered Beneficiary Details' screen where you can store additional information of your Registered Beneficiaries like their personal details, bank details, account details, contact details, etc. Refer to the Beneficiary master for the same.

Personal Details

□         Beneficiary Code: Type a unique code that will be used to identify and reference the beneficiary.

□         Name: Type the name of the beneficiary

□         Mobile Number: Type the beneficiary mobile number.

□         E-Mail ID: Type the beneficiary e-mail address.

□         Resident Status: Select the residential status of the beneficiary. The available options are Resident (default) and Non-Resident.

 Beneficiary Bank Details

□         Bank Type: The options are Registered Bank (checked by default), and Adhoc Bank. This is dependent on your settings in the My Product master.

□         Bank Branch Search: Click on the Seek icon to select the Bank Branch code.

-         In case of registered bank and depending upon the set up done by the bank, bank and branch name in the seek list will show the names in local language also.

-         In case of Adhoc bank, the user needs to input the Bank and Branch name in English.

□         The Bank and the Branch fields get auto populated and uneditable. These fields are displayed in English as well as local language if maintained in local language by the bank.

□         BIC: Type the beneficiary BIC code (SWIFT), in case of cross-border funds transfer.

□         Bank Address: Type the address of the beneficiary bank.

□         Bank Country: Type the name of the country in which the beneficiary bank is located.

Click Swift Details button to open the below popup and enter the Swift message. Click Close to close the box.

Beneficiary Account Details

□         Account Number/IBAN: Type the account number of the account that the Beneficiary holds in the specified bank branch or the Beneficiary's IBAN number..

□         Account Type: Select the type of account that the Beneficiary holds from the drop-down list. The options are NRE, NRO, Current, Saving and Loan

□         Account Currency: Select the Currency in which the Beneficiary’s account is held, from the drop-down list displaying a list of currencies.

Adhoc Beneficiary

Displayed if Adhoc Beneficiary is specified for the MyProduct. Click the Adhoc Beneficiary tab to open the Adhoc Beneficiary pop-up screen and enter the details pertaining to the beneficiary, along with the contact details and the Intermediary Bank details.

Payment Details

□         Processing Date: Mandatory. This field is defaulted to current application date. You can select a future date but not a past date

□         In case of Cross Border Payment Transactions, i.e. when Debit Account and Payment Currencies are different,fields Rate Type, Indicative Rate, Contract Reference Number are added to the screen.

□         Rate Type: This field is enabled for cross currency transactions, i.e. when debit account and credit account currencies are different. This field indicates the type of conversion rate to be applied to the transaction for converting cross currency transactions. The available options are:

•        Card Rate: This is the spot rate which is obtained from the “FX Rate Entry” screen from Bank end. Current rates are uploaded in this table at schedule frequencies by the system. By default, this field displays the Standard Rate.

Note: In case Deal Rate is selected, an additional validation is introduced to check for Client threshold in Payment transactions (Payment FX Threshold). System internally converts the payment amount in seller currency using latest card rate. If the payment amount equivalent is above the ‘Payment FX Threshold’, system displays a message - ‘Transaction Amount is above Threshold, Get Deal for payment processing’. In the above scenario system will automatically change the Rate Type value as ‘Deal Rate’ and display the (Get Deal) button.

•        Deal Rate: The value in this field is obtained via the Get Deal Rate Interface provided in the Interfaces User Guide. This rate is defined for a particular period.  If the user selects ‘Deal Rate’, it is mandatory to provide the Deal Reference Number.

□         Indicative Rate: This field displays the indicative rate using which the debit amount is converted to the payment amount or vice versa depending on the radio button selected.

□         Payment Amount: Select the radio button if you want to specify the payment amount. The Payment Amount reflects the Payment Currency selected. Type the transactional value of the instrument. This amount is automatically derived in case you select Debit Equivalent radio button and enter the amount in that field.

□         Debit Equivalent: Select the radio button if you want to specify the debit amount and type the transactional value of the instrument. This amount will be used to debit the client in the currency in which the debit account is maintained. This amount is automatically derived in case you select Payment Amount radio button and enter the amount in that field

□         Contract Ref No.:This field is editable only when ‘Contract Rate‘ is selected in the above field. Click Seek. The “Contract Reference Help” popup screen appears displaying the Contract Reference Codes, FXOnline Deal Reference Codes, Contract Rates, From/To Amounts and Deal Client Ids of the contracts that are defined for the logged-in client by the system bank. Select the appropriate Contract Reference or the  FXOnline Deal Reference for the currency pair.

□         Transaction Charges: Once the Debit Amount or the Payment Amount is specified, system calculates the transaction charges and displays it in the below field.

□         Purpose Code:  Click to select the purpose code. This field provides a seek-list of the appropriate purpose code associated with the instrument code depending upon the Bank payment product selected by the user. If no Bank Payment product is selected then the Seller level Purpose Codes maintained by the Seller at the bank are displayed in the list.

□         Purpose Code Description:  This field displays the description of the purpose code selected. In case Purpose code is selected as OTHER, then the you can enter the description as required.

Processing Details

□         Instrument Date: Displayed if mode of payment is cheque and defaulted to current application date.

Note: Can be future dated or backdated up to 6 months from the current application date.

□         Charge To: Select from the available options- Client, Beneficiary or Shared

□         Payment Location: Displayed for physical products Demand Draft / Cashier Order / Customer Cheque. You can select a Payment Location from the list maintained by the bank.

The 'drawn on' locations where the beneficiary would like to receive the payment. Payment location would be defaulted to client branch location, if user selects bank product as Cashier Order in which case this field would be non editable. In case of other products system will show all client product clearing locations or a list of drawee bank branches for the selected UDE (Universal Data Entry) product. In case the bank product is Cheque, select the drawee bank branch where the cheque is issued on. The Service Branch of the payout location is the branch on which the cheque would be issued.

□         Pickup Branch: Displayed for Demand Draft / Cashier Order / Customer Cheque where delivery mode is "Collect From Branch". You can select a Payment Branch from the list maintained by the bank.

□         Central Bank Reference: This number is reflected in view mode of the transaction after the bank updates the details at their end.

Delivery Details

□         Delivery Mode: Available options (Collect from Branch, Send to Beneficiary Bank, Mail to Customer, Courier to Customer) - Default - Collect from Branch or based on the setting in the Beneficiary master

□         Delivery Branch: If delivery mode 'Collect from Branch' then Delivery Branch mandatory. Displays seek list of System Bank Branches (Default - Client Controlling Branch)

□         Authorized Person: If delivery mode 'Collect from Branch' then  Authorized Person is mandatory.

□         Authorized Person ID: If delivery mode 'Collect from Branch' then  Authorized Person ID is mandatory.

Beneficiary Notification Details

□         Benef E-mail: Type the email ID of the beneficiary. This field is mandatory if you select EMAIL option in the Advice Delivery Mode field. You can type multiple email IDs separated by a comma.

Additional Information

□         Debit Reference: Type the reference number of the debit entry

□         Debit Details: Type the debit details or any additional information for the debit entry.

□         Credit Reference: This field is displayed for Electronic Payment Products (IFT, NEFT and RTGS). Specify the Credit reference.

□         Credit Details: Type the credit details or any additional information for the debit entry.

□         Supp. Doc Name: Enter the supporting document name in this field.

2.      Click to save the instrument details.

3.      In case Advice Information fields are pre defined for the Client Product combination, the Advice Info. tab is enabled and entering the details is mandatory. Click the Advice Info. tab to open the Payment Advice Info. screen and click Add to add the values in the advice information fields defined for the product. Save the advice information and click Add to add more values.

Note: Every advice information needs to be saved before you can add the next one.

4.      Click Save to save the Advice Information, and then click Back to exit from the Advice Info. entry screen to view the newly entered Advice Information in the Advice Info. summary screen.

Tip: In case of instrument entry for Cash transactions, you can enter denominations in the tab Denominations which appears after save. If denominations are maintained in local language by the bank then these are available in the local language also.

5.      Click the Instrument tab to go to the 'Instrument' entry screen and click Save to save the instrument. A message indicating that the instrument is successfully saved appears on the screen. At this stage, you can delete the instrument by selecting it and clicking   that appears under More.

6.      In case of any errors, you can also Edit the instrument and click to update the transaction.  (To add more instruments, click 'Add Instrument'  in the Batch Header and then repeat the same process).

7.      Click against an instrument to view the instrument details. View screen will reflect the Central Bank Reference number added by the banks for the instruments that are processed at the bank end.

8.      Click Back to go to the Batch Header screen where you can view the newly added instrument under Instrument Details section. The Submit button is enabled which indicates that the transaction can be submitted for authorization. You also have the option to Discard it.

Note: If the instruction that you have entered is the last instruction within the payment transaction, system performs the checksum validation and notifies the discrepancies, if any. You need to correct these discrepancies before you close the payment transaction.

9.      Click the Back button till you go back to the Payment Center screen.

In case you are adding Instrument level for a Standing Instruction- Batch Payment, click Standing Instruction Center for further processing of the transaction.
