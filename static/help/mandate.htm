Mandate List

Collection >> Masters >> Mandate

Mandate is the authority given by the payee to the receiving party (in our case the GCP CashWeb Client) to initiate a payment on behalf of him. The client user  can also maintain the mandates of the drawer through GCP CashWeb. The client can maintain the mandate submitted by the drawer manually or through file upload. The direct debit transaction initiated by the client using drawer mandate is validated against the mandate maintained in this master. The Mandate Code is always unique.

The Mandate List screen shows a summary of the existing Mandates received for the Payers.

View the details under the following heads

□         No.:This is the Serial Number

□         Enable: If you want to use the disabled definition again in the system, you need to enable it. If accepted on authorization, the definition will be enabled. If rejected on authorization, the rejected request will have to be discarded so as to revert the definition to its Disabled state.

□         Disable: You can disable a definition, if it is not to be used in the system any more

□         Discard: If you have created definitions that you do not want to enable for further use in the system and if those definitions are not yet authorized, you can delete them. Similarly, you can delete the new definitions that are rejected in authorization.

□         Mandate Code: This unique code is used to identify and reference the Mandate.

□         Description: Type the description of the Mandate

□         Status: Every definition is associated with a state that indicates the current action that has been taken on it.

Enter the Mandate Code and click Filter to retrieve a particular Mandate definition.

Defining a Mandate:

§        Click Add New Mandate to go to the Mandate Entry screen to set up the new Mandate.

§        A successfully defined Mandate appears in this screen with status as 'New' and if needed you can Edit or Discard the Mandate.

§        A successfully authorized Mandate will also appear in this screen with the status - 'Authorized' and you can Enable or Disable it provided you have the rights to do so

§        If you are eligible for authorization, the record will be available in the Mandate Authorization List screen and you can then Authorize the same.

Note:  

-         If you have already set up the accounts of a Mandate that you do not want to use in future and if the Mandate definition is not yet authorized, you can discard such a definition. Discarding an unauthorized Mandate automatically deletes all the account definitions set up for the Mandate.

-         To add more Mandates, click Add New Mandate and then repeat the same process

Editing a Mandate Definition

1.      Click Edit to go to the Mandate Entry screen where you can edit a previously defined master.

2.      Click Update to save the changes.

Note:

 You can modify the definitions if they are in any of the following states:

-         New definition: Any new definition is not yet authorized and is in the new state.

-         Modified definition: Any definition that you have modified and not yet authorized and is in the Modified state.

-         Enabled definition: The definition that is authorized and is in the Enabled state.

The edit option is not available for the definitions that have the state as Disabled, Disable Req, Enable Req, Dis. Req. Reject, and Ena. Req. Reject, Submitted, Authorized.

On saving the modifications, the definition state changes to Modified. To enable the definition with the Modified state, the definition has to be authorized.

Viewing a Mandate Definition:

1.      Click View to go to the Mandate Entry screen where you can view a previously defined master. You cannot modify the definition in this screen.

2.      Click Back to return to the Mandate List screen.
