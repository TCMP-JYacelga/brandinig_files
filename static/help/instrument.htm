Instrument  

Adding Instruments to a Deposit

1.      In the Instrument Entry screen, initiate an instrument-level deposit request.

•    Specify the following Instrument details:

•    Instruction Details

□         Collection Currency: This field reflects the transaction currency entered during Batch entry and is not editable.

□         Collection Product: Mandatory. This field may appear either at Deposit level or at Instrument level depending on the My Product setup. Select the collection product for the specified MyProduct transaction from the drop-down list displaying all the valid collection products.

□         Reference: Mandatory. Type a unique reference number. This field displays the value specified in the Deposit Reference field at the Deposit header level and is not editable

□         Collection Product: Mandatory. Auto-populated based on Deposit screen values

□         Account Number: Mandatory. Auto-populated based on Deposit screen values.

□         Collection Currency: Auto-populated based on Deposit screen values

•    Payer Details

□         Payer Code: Mandatory. Select the Payer Code from the seek list displaying a list of your regular drawers as set up in the Payer master. On selecting a drawer, its relevant details are automatically populated in the rest of the fields. However, you can change the values.

□         Mandate Code: Select the Mandate Code against which you are receiving the payment, from the seek list. This field is mandatory only in case of Direct Debit products, else hidden

□         Payer Name: This field is auto-populated  based on the Payer Code entered.

□         Adhoc Payer: Click this button in case of Adhoc Payer and type the Payer details in the Adhoc Payer Entry screen in the respective fields.

More Details

□         Drawee Bank Code: Auto-populated based on Mandate Code entry. In case of Adhoc Payer, it will be populated from Adhoc Payer details entered.  

□         Drawee Branch Code: Auto-populated based on Mandate Code entry. In case of Adhoc Payer, it will be populated from Adhoc Payer details entered.  

□         Payer Account Number: Payer Account Number is mandatory in case of Direct Debit else hidden, and is populated based on Mandate Code selected. In case of Adhoc Payer, it will be populated from Adhoc Payer details entered.

•    Amount

□         Instrument Amount: Type the instrument amount for the deposit.

Note: If the Deposit contains only one instrument, the value that you enter in this field must be equal to the value that you have entered in the Total Amount field of the Deposit header details. Otherwise, the amount has to be lesser than the Total Amount.

□         Currency: This field displays the currency in which your account is held and is non editable.

□         Activation Date: Select the activation date in DD/MM/YYYY format. This is the date when processing of the collection instruction begins. The activation date has to be a date same as or subsequent to the application date. The display of this field depends on the level set in the Template menu of My Product Master Entry.

□         Purpose Code: Values will be defaulted from the Mandate Master. You can modify these by clicking to select the purpose code. This field provides a seek-list of the appropriate purpose code associated with the instrument code depending upon the Bank payment product selected by the user. If no Bank Payment product is selected then the Seller level Purpose Codes maintained by the Seller at the bank are displayed in the list.

□         Purpose Code Description:  This field displays the description of the purpose code selected. In case Purpose code is selected as OTHER, then the you can enter the description as required.

•    Notification details

Specify the drawer notification details:

□         E-Mail: This field will display the default email id as setup in Payer master in case of registered Payers.

In case of Adhoc Payers, you can specify the email id.

□         SMS: This field will display the default mobile numbers as setup in the Payer master in case of registered Payers.

□         In case of Adhoc Payers, you can specify the Mobile numbers.

•    Additional Collection Information

□         Credit Reference: This field is displayed for Electronic Payment Products (IFT, NEFT and RTGS). Specify the Credit reference.

□         Credit Detail: Type the credit details or any additional information for the credit.

□         Debit Reference: Type the reference number of the debit

□         Debit Detail: Type the debit details or any additional information for the credit.

2.      Click to save the instrument.

3.      In case Advice Information fields are pre defined for the Client Product combination, the Advice Info. tab is enabled and entering the details is mandatory. Click the Advice Info. tab to open the Deposit Advice Info. List summary screen. Click Add Advice Info button to open the Deposit Advice Info. Entry screen to add the values in the advice information fields defined for the product.

Note: Every advice information needs to be saved before you can add the next info.

4.      Click Save to save the advice information, and then click Back to exit from the Deposit Advice Info. Entry screen to view the newly entered value in the Deposit Advice Info. List. summary screen.

5.      Click the Instrument tab to go to the 'Instrument' entry screen and click Save to save the instrument. A message indicating that the instrument is successfully saved appears on the screen. At this stage, you can delete the instrument by selecting it and clicking   that appears under More.

6.      In case of any errors, you can also Edit the instrument and then Update it.

7.      Click the Back button  to come to the Deposit screen (To add more instruments, click 'Add Instrument'  in the Deposit header screen and then repeat the same process).

Note:

-         If the instruction that you have entered is the last instruction within the Deposit, system performs the checksum validation and notifies the discrepancies, if any.  An error message is flashed on the screen with the relevant message. You need to correct these discrepancies before you close the entry.

-         System will validate the total instruments and their amounts with the entered instruments & amounts at header level. In case of any discrepancies, system will display an appropriate error message. This validation is performed based on the Zero Proofing flag set at My Product level.

-         In case the My Product flag is set to Instrument for Submit, Authorize and Send, the same will have to be done for individual instruments at instrument level in the Deposit screen.

8.      Click Back come to the Collection center screen to Submit, Authorize and Send the transaction. A successfully saved transaction will appear in the Collection center with status as  Pending Submit. You have the option to Discard  a transaction till such time that it is authorized.
