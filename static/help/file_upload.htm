Uploading Multiple Transactions

Payments >> Payments >> File Upload

About File Upload

This feature enables you to upload multiple transactions via a single file in the system. It reduces the efforts required for manually entering the bulk transactions through data entry screen. On uploading the file, GCP CashWeb carries out file format validations followed by data level and field level validations. Once the transactions are successfully uploaded, they are automatically closed and appear in the authorization screen. Auto authorization and auto release depends upon the parameters set in My Product master.

Client can also upload the file generated by the ERP system directly into the GCP CashWeb using this facility. The application at a time supports upload of transaction file for a given Client and MyProduct combination. MyProduct can include different payment products such as internal fund transfer, Low value electronic, High value electronic, Cross border transfer, Managers cheque etc based in UDE product linkage to MyProduct and its respective product association.

Note:

-          In case advice infos are defined for MyProduct then they will be validated and populated during file upload against the MyProduct advice info field else against the client product advice ifno fields. If the advice infos are not defined for MyProduct and Client Product, then the advice info values uploaded in the file will be ignored.

-          In case the advice infos are mandatory and yet the file is uploaded, the same will appear with status- ‘For Repair’ in the Payment summary screen. Select the same and enter the advice info as defined for that product.

-          For automatically closing the uploaded transactions, the Close Required flag must not be selected in the My Product master.

-          For auto authorization and auto release, the Approval Required & Send Required flags must not be selected in the My Product master.

The File Upload Center displays a summary of previously uploaded files along with their related information and status . The display is refreshed every few minutes, to provide you with the most updated information.

□         File Name: This field displays the unique file name without the extension

□         Date and Time: This field displays the date on which the multiple payment file is uploaded.

□         Status: This field displays the status of the uploaded file. The different states of upload can be: New, In Queue, Running, Aborted, and Completed.

□         Remarks: This field displays any remarks that may have been generated by the system, indicating the state of upload. In case, any errors occur during the upload process, relevant error messages will be displayed to you in this field. This will help you to identify the problem with the uploaded file.

Uploading Multiple Transactions:

1.      Click the icon in the File Upload Center to open the 'Upload File' screen

2.      Enter the following details:

§        File Format Type: Select the Upload File Format Type from the drop down list. This list comprises of all the preloaded file formats available for the client.

§        File Name: Select the file from your network, using the Browse option. If you are entering the name of the file, ensure that you are specifying the absolute path of the file.

3.      Click Upload to upload the selected file. GCP CashWeb will run file level validations as well as data level validations on the file. In the course of validating an uploaded file, if system encounters any errors that can be repaired, the uploaded file is moved to 'Reject Repair'. If the errors cannot be repaired in ‘Reject Repair’, then system aborts the filer. This triggers an event and a notification of the reject reason is sent to the Client as well as the Bank User by E-mail or SMS as specified in the Event Notification master. The reject report is available for viewing or downloading in the File Upload summary screen. If both field and data level validations are successful, system uploads the file successfully and GCP CashWeb displays the message “File Uploaded Successfully”.

OR

Click Cancel to cancel the upload request.

IMP:

-         The maximum file size allowed for upload is 5MB.

-         In the course of validating an uploaded file, if system encounters any file related errors, the same is aborted. In case of data level errors, the file is sent for Reject Repair. This triggers an event and a notification is sent to the Client as well as the Bank User by Email or SMS as specified in the Event Notification master. The same can also be viewed on-screen in the File Upload summary screen. You can also view or download the Reject report. The rejected file appears in the Payment center with status -Pending Repair and  user can repair the errors and validate the file for further processing.

4.      Once the transaction is uploaded successfully, it appears in the Payment Summary screen with status as Draft with the option to Discard it. Here on the same workflow steps (Authorization and Send), as in the case of Manual Data Entry have to be followed. If Auto Auth and Auto Send functionality is enabled in the MyProduct master, the transaction will directly appear in ‘Sent to Bank’ status.

Note:

-         For automatically closing the uploaded transactions, you have to set the Auto Submit flag as checked in the Payments My Product Master Entry >> Workflow >> File Upload.

-         In case of auto authorization and auto release, you need to set the Approval Required & Send Required flags unchecked in the Payments My Product Master Entry >> Workflow >> File Upload.

Viewing the Reject Report

In case an uploaded file gets aborted due to file level validation failure or appears in Reject Repair due to data level validation failure, you can download the Reject Report to understand why the file was aborted or rejected.

1    In the File Upload screen, click the Download hyper link of a file under the 'Reject Report' column  which is either aborted or rejected.

2    In the File Download popup that appears, click Open to view the generated report or you can Save it on your local machine in .pdf format.  

Repairing Rejected Transactions

When you upload a file with multiple transactions, GCP CashWeb performs all the validations that are performed at the time of data entry. If the system encounters any errors while uploading the file, you can edit or repair the errors at instrument level and send it for further processing.

1    In the Payments Center, click the Instrument tab to open the instrument summary screen.

2    From the Status drop down, select Pending Repair to retrieve all the instruments that are available for repair

3    To repair an instrument, click  Edit Instrument . The Instrument Edit screen opens displaying the error reason in the header area.

4    Make the necessary changes. System refreshes the screen with the updated value and the error displayed on the screen disappears.

5    Click Update to save the changes, and then Back to go to the Payment center.

6    The status of the repaired transaction changes to ‘For Submit’ and the Submit button gets enabled.

7    Click Submit and the status of the transaction changes to ‘For Auth’.

8    To Authorize a repaired transaction, click Authorizing a Payment Transaction and Releasing it to the Bank

□         Charge To is Mandatory

□         Value Date is Mandatory

□         If the file contains any of the blacklisted characters and Oracle SQL keywords (like Insert/Update/Union etc) file will be outright rejected.
