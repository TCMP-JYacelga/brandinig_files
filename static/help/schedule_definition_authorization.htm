Schedule Definition Authorization

Reports >> Reports >> Report Schedule Auth

All report schedule definitions require authorization. A scheduled report will not be generated unless it has been authorized and the same has to be authorized following the process of the Maker-Checker.

IMP: You must select an appropriate client from the Corporation in order to authorize a schedule definition through the  Switch Client option.

The Schedule Definition Authorization screen displays all the unauthorized report schedule definitions.

You can filter the search by Report Code. The drop-down list shows all the report codes specified in the system.

Authorizing a Schedule Definition:

§        Select the Accept check box of a Schedule Definition that is awaiting authorization in various states like 'New', 'Disable Request', 'Enable Request', etc.

§        The authorized definition moves out of this screen and can be viewed in the Schedule Definition List screen with the status as Authorized, Enabled or Disabled.

§        Select the Reject check box in case you want to reject a schedule. While rejecting a schedule for a report, it is mandatory to enter the reason for rejection.

A schedule that is accepted on authorization is an enabled schedule, the same can be disabled. Likewise, you can also enable a disabled schedule. Disabled, and enabled schedules also require authorization. If a request for disable, or enable is rejected on authorization, then the request must be discarded. On discarding, the schedule is reverted back to its original state.

For further details, refer to Other Possible Tasks.

Viewing a Schedule Definition:

1.      Click View to go to the Schedule View screen where you can view a previously defined schedule. You cannot modify the same in this screen.

2.      Click Back to return to the Schedule Definition List screen.
