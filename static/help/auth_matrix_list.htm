About Authorization Matrix

□         Any transaction originated in the system needs to be approved by one or more responsible users of the system.

□         Authorization Matrix defines the number of the times a record will have to be approved or verified before it becomes active. In other words, the Authorization Matrix defines the verification and approval rounds required for any request or activity performed on a transaction. This ensures that the transaction passes through the approval process to safeguard it against any deviation or fraud.

□         The Authorization Matrix is defined based on either the MyProduct or the Account master, as per the value set in the MyProducts master. In this master, for each product or account you need to set up various limits. For each limit, you need to the assign the users who are required to approve the transaction.

IMP:

-         Before you set up the authorization matrix master, ensure that you have set up the product/account masters.

-         Ensure that you select the client for which you want to set up the master definition using Switch Client.

The Authorization Matrix List screen displays all the authorization matrices that you have defined in the system, along with their state of authorization. Against every definition are options to delete, enable, or disable that record.

You can filter by Authorization Master and Product to retrieve a particular definition.

Defining an Authorization Matrix:

§        Select the value in the Authorization Master field. The options are Authorization Matrix and Signatory Matrix.

§        In case you select Authorization Matrix, click Add to go to the Authorization Matrix Entry screen.

§        In case you select Signatory Matrix, click Add to go to the Signatory Matrix Entry screen.

The newly defined Authorization Matrix is added to the Auth Matrix List, with its state specified as New and the same can be edited or discarded.

To enable the Authorization matrix, it is mandatory to authorize it based on the Maker-Checker concept. The checker can view the new definition in the Authorization Matrix Authorization List screen and authorize the same provided the rights to authorize are assigned to that checker.

A successfully authorized Authorization Matrix definition request can be viewed in the Auth Matrix List screen with its status as Authorized. Against every definition are options to delete, enable, or disable that record.

Note:  If you have created authorization matrix definitions that, you do not want to enable for further use in the system, and if those records are not yet authorized, you can delete them.

Editing an Authorization Matrix Definition:

1.      Click Edit to go to the Authorization Matrix Entry screen where you can edit a previously defined master.

2.      Click Update to save the changes.

Note:

  You can modify the definitions if they are in any of the following states:

-         New : Any new definition is not yet authorized and is in the new state.

-         Modified : Any definition that you have modified and not yet authorized and is in the Modified state.

-         Authorized: The definition that is authorized and is in the Enabled state.

The edit option is not available for the definitions that have the state as Disabled, Disable Request, Enable Request, Disable Request Rejected, and Enable Request Rejected, Submitted, Authorized.

On saving the modifications, the definition state changes to Modified. To enable the definition with the Modified state, the definition has to be authorized.

Viewing an Authorization Matrix Definition:

1.      Click View to go to the Authorization Matrix Master View screen where you can view a previously defined master. You cannot modify the definition in this screen.

2.      Click Back to return to the Authorization Matrix List screen.
