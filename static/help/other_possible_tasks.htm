Other Possible Tasks

Viewing History of a Definition:

Click the History icon to view history details of the product record. The history details show the Maker-Checker history for the last change done on the record.

Cloning a Definition:

Click the Clone icon to copy the record details to a new  record. All the details pertaining to that transaction get populated in the transaction entry (Batch/Deposit Header) screen. You can modify the fields as per your requirement and follow the procedure similar to 'Transaction Initiation'. This feature is applicable only to Transaction Entry

Discarding a Definition:

If you have created definitions that you do not want to enable for further use in the system and if those definitions are not yet authorized, you can delete them. Similarly, you can delete the new definitions that are rejected in authorization

Discarding a Rejected Request for a Master Definition:

If a Disable Request, Enable Request, or Modified Request for a definition is rejected on authorization then this definition has to be reversed to its original state. You can do this by discarding the rejected request for the definition.

The Definition Entry List tells you at a glance whether a particular definition has been rejected. This is indicated in the State column, which displays the current state of the corresponding definition. Depending on the request that is rejected, the State column takes different values.

Disabling a Master Definition:

You can disable a definition, if it is not to be used in the system any more. Clear the Enable check box against the definition that is to be disabled. The value in the Status column will change to Disable Req. This implies that the change in the state needs to be authorized.

§        If accepted on authorization, the definition will be disabled.

§        If rejected on authorization, the rejected request will have to be discarded to revert the definition to its Enabled state.

Enabling a Disabled Master Definition:

If you want to use the disabled definition again in the system, you need to enable it. Select the Enable check box against the disabled definition that is to be enabled. The value in the Status column will change to Enable Req. This implies that the change in the state needs to be authorized.

§        If accepted on authorization, the definition will be enabled.

§        If rejected on authorization, the rejected request will have to be discarded so as to revert the definition to its Disabled state.
