User Authorization List

Admin >> User Setup Auth >> User Auth

To enable the User, it is mandatory to authorize it based on the Maker-Checker concept. The checker can view the new definition in this screen and authorize the same provided the rights to authorize are assigned to that checker.

To Authorize a User:

§        As Checker, select the Accept check-box to authorize the request.

You can filter the search by User Code to retrieve a particular definition.

§        You may also select the Reject check box to reject the request. While rejecting a request, it is mandatory to type the reason for rejection. Type the reject reason and click OK.

§        On authorization, the User definition moves out of this screen and can be viewed in the User List screen with its status as Authorized.

 

Viewing a User:

1.      Click View to go to the User Master Entry screen where you can view a previously defined master. You cannot modify the definition in this screen.

2.      Click Back to return to the User List screen.
