Category List

Admin >> User Setup >> User Category

A Category is a set of access rights which can be attached to a group of users performing similar tasks. The same also can be used to control the use of the various menus in the GCP CashWeb. Defining categories facilitates segregation of tasks to be performed by various groups thereby ensuring the smooth functioning of the organization

Before, creating a user ID for your client, you have to create a category to which the client user will belong. While creating a category, you can assign access rights to the category based on the internal structure and hierarchical levels of your client’s organization. Once you create a category, it has to be authorized so that you can add users to the enabled category

Defining a Category

Defining a category would mean assigning a unique identification and relevant access rights. You can also assign rights based on the internal structure and hierarchical levels of your client’s organization. To activate the category definition, it is essential to authorize the definition.

IMP: Ensure that you select the client for which you want to set up the  definition using Switch Client.

1.      Click Add Category to go to the Category Master Entry screen to add a new category for your client user.

2.      The newly defined category is added to the Category List, with its state specified as New and the same can be edited or discarded.

3.      To enable the category, it is mandatory to authorize it based on the Maker-Checker concept. The checker can view the new definition in the Category Authorization List screen and authorize the same provided the rights to authorize are assigned to that checker.

A successfully authorized Category definition request can be viewed in the Category List screen with its status as Authorized.

Note:

-          If you have created categories that you do not want to enable for further use in the system and if those categories are not yet authorized, you can Discard them.

-         You can modify category information by altering the rights that have been assigned to it. To modify a category, either the existing rights can be revoked or new ones can be granted. However, any changes made to the existing rights of a category will not automatically apply to the users within that category. The renewed rights will have to be applied to each user within that category to maintain consistency, if required.

-         As a user of the application, you will not be able to edit information of the category to which you belong.

-         To disable a category, the users enlisted in that category must be disabled first.

-          If you enable a category, the user belonging to the category before disabling it, will not be enabled automatically. You can enable these users manually.

Editing a Category:

1.      Click Edit to go to the Category Master Entry screen where you can edit a previously defined master.

2.      Click Update to save the changes.

Note:

 You can modify the definitions if they are in any of the following states:

-         New : Any new definition is not yet authorized and is in the new state.

-         Modified : Any definition that you have modified and not yet authorized and is in the Modified state.

-         Authorized: The definition that is authorized and is in the Enabled state.

The edit option is not available for the definitions that have the state as Disabled, Disable Request, Enable Request, Disable Request Rejected, and Enable Request Rejected, Submitted, Authorized.

On saving the modifications, the definition state changes to Modified. To enable the definition with the Modified state, the definition has to be authorized.

Viewing a Category:

1.      Click View to go to the Category Master View screen where you can view a previously defined master. You cannot modify the definition in this screen.

2.      Click Back to return to the Category List screen.
