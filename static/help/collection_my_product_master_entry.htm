Collection My Product Master Entry

This screen allows you to associate the Bank products offered to your own Collection My Product and also define attributes and workflow rules like authorization pattern, user access rights, authorization type such as transaction or PIR etc.

1.      To add a new  Collection My Product, type the necessary information in the fields provided under the following heads:

§        My Product

Product Details

□         Product Code: Type a code that will uniquely identify the My Product across the system.

□         Product Description: Type a description for the product

Note: The system does not support the following special characters: “ \ ~ ^ and ‘ in above text fields

□         UDE Bank Product: Select the corresponding product from the drop-down list. This step maps your product with the product defined at the bank.

□         My Product Usage: Select the mode of collection for the specified My Product. Options are:

QuickColl: This mode of collection means 1 Batch 1 Instrument.

Batch Collection: This mode of collection means 1 Batch Many Instruments

Configuration Details

□         Authorization Type: This field determines whether the approval for the specified product must be based on Product or Account. The available options are:

My Product: Select this option if you want the approval to be Product based.

Account: Select this option if you want the approval to be Account based.

Maker Checker: Select this option if you want to approve the foreign currency transactions.

□         Authorization Level: Select 'Instrument Level' in case you require authorization for each individual instrument. Select Batch Level for authorization at batch level.

Adhoc Details

□         Adhoc Drawer: Select the 'Allowed' check box if you want to allow Adhoc Drawer in the transaction initiation for this MyProduct. If you do not select this option, only registered beneficiaries can be involved in the transactions of this product. By default, this option is enabled

□         Adhoc Drawee Bank: Select the 'Allowed' check box if you want the Adhoc Drawee Banks to participate in the transactions. If you do not select this option, only regular beneficiary banks can be involved in the transactions of this product

§        Workflow

Manual Entry (Enables clients to bring more flexibility to the product definition by controlling the authorization and send process)

□         Approval Required: Select the check box if you want to manually authorize the manually entered transactions.

□         Send Required: Select the check box if you want to route the manually entered transactions through the sending option after authorization.

File Upload

□         Submit Required: Select this check box to allow the transactions to be submitted on Save.

□         Allow Transaction Edit: Select this check box to allow uploaded transactions to be edited.  

□         Approval Required: Select the check box if you want to manually authorize the uploaded transactions.  

□         Send Required: Select the check box if you want to route the uploaded transactions through the sending option after authorization 

□         Batch Upload Action: Select the reject repair process from the list. These are based on the system parameter configured at the client level. The available options are:

  1. Process All Records in Reject Repair

    Process OK Records.

    Reject Entire File on Error.

    Process OK records with balance in Reject Repair

Validations

□         User Product: Select this check box to generate a relation between the user and the specified product. If you select this check box, the user can view this product at the time of data entry. By default, this check box is selected  

□         Zero Proofing: Select this check box if you want to specify the total number of instruments and total amount at the PIR header level for that product. If you do not select this check box, the total number of instruments and total amount fields are disabled at PIR header level. The number of instruments that can be specified at PIR would be automatically set based on Transaction captured at Instruction level. By default, this check box is selected 

□         Scrap if not Auth or Send: Maker can specify the number of days after which transactions that are unauthorized or not sent to bank for further processing will be scrapped.  

□         Notify User of Scrap before:  Specify the number of days before which the user will be notified about a definition being scrapped

§        Template

Additional Workflow Details

□         Collection Product: Select the default Collection product to be viewed at Instrument Level or Batch Level.

□         Credit Account: Select the default Credit Account to be viewed at Instrument Level or Batch Level    

□         Activation Date: Select whether activation date should be at Instrument Level or Batch Level.   

Additional Notification Details

□         Activation Date:  Select the check box to make the Activation Date mandatory for the selected product

□         Email Notification: Select this check box in case email notification is to be made mandatory for the user.

□         SMS Notification: Select this check box in case SMS notification is to be made mandatory for the user.

□         Advance Notification: Select this check box in case advance notification is to be made mandatory for the user.     

□         My product Advice Info: This section displays the deposit and instrument level advice information details. Select the check box against the Advice Info that will be mandatory for the transaction and specify the info label in the subsequent box. You can specify up to five information labels each at Deposit and Instrument level.

2.      Click Save to save the entry and return to the Collection My Product List screen where you can view the newly entered My Product with status as New. This implies that this is a new addition and yet to be authorized and the same can be edited or discarded.

3.      The Collection My Product Master definition is added to the list in the Collection My Product Authorization List  screen with its status as New, where it awaits authorization.

4.      Click Back if you want to go to the previous page without defining the product.
