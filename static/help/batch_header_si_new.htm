Batch Header SI

The Batch Header screen allows you to maintain a summary level record of a batch of multiple instructions.

The header displays the My Product selected and the Internal reference Number

1.      Enter the Batch Header Details

The fields will vary depending on the selected Product and are similar to the Payments Batch Header

The fields specific to Standing Instructions setup are as below and are required for setting up the frequency for executing the Standing Instruction.

□         SI Frequency: Select the frequency that you want to set up for the standing instructions from the drop-down list.

□         StartDate:Select the Effective Date from the calendar pop-up. The field gets populated with the start date in DD/MM/YYYY format, i.e. the date from when the frequency will be activated. Ensure that the Effective Date is greater than the application date.

□         End Date: Select the Termination Date from the calendar pop-up. The field gets populated with the end date in DD/MM/YYYY format i.e. the date till when the frequency will be valid. Ensure that the Termination date is always more than Effective Date. The default value for this field is- 'Till Terminated'.

□         No. of Times: Once you enter the SI Frequency, the Effective Date and the Termination Date this field automatically shows the number of times the payment will be processed.

□         Holiday Action: Select the action that should be taken in case the transaction date falls on a holiday as defined by the bank in the Holiday Setup.. The available options are- Postpone which is the default value, and Prepone.

For rest of the fields and further processing, refer to Payments- Batch Header.

Note:  Once the setup is saved, system will automatically initiate the SI transaction based on the specified frequency.

2.      Click icon to save the payment information request. The message - 'Payment Information Request Successfully saved' along with the Batch Reference Number is displayed. The Internal Reference Number is also generated by the system.

3.      In case you modify the data in any of the editable fields, click to save the changes.

4.      Click Add Instrument to go to the  Instrument Entry screen to add the instrument level details.

Note: Once the instruments are added, system will validate the total instruments and their amounts with the entered PIR instruments & amounts at header level. In case of any discrepancies, system will display an appropriate error message. This validation is performed based on the Zero Proofing flag set at My Product level.
